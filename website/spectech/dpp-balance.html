Проверка данных

Получение и обработка данных первичного производственного учёта. Проверка и фильтрация первичных измерений для предоставления достоверных и надёжных данных.

Согласование данных

Использование избыточности информации для корректировки каждого измерения в целях удовлетворения производственных балансов. Позволяет выявлять неисправные приборы учёта.

Математическая модель ДПП.Балансы основана на универсальных условиях баланса массы для заданного набора технологических объектов. Система поддерживает неизмеряемые потоки и ограничения по метрологической погрешности.
Постановка задачи

Имеются измерения y(yj) с допустимыми погрешностями σ(σj). Требуется найти скорректированные значения x(xj), удовлетворяющие балансовым уравнениям при минимальных поправках.

Ax = c    (балансовые уравнения)
Σ (yj - xj)² / σj² → min    (критерий минимальных поправок)
0 < xmin ≤ xj ≤ xmax    (ограничения по уставкам)
|yj - xj| ≤ σj    (метрологические ограничения)
Методы решения
Все потоки измереныx = y − WAT(AWAT)−1(Ay − c)
Есть неизмеряемые потокиQR-разложение матрицы Au для построения матрицы проекций P, затем решение для измеряемых и неизмеряемых компонент
С ограничениямиИтерационный процесс: решение → проверка ограничений → коррекция → повтор до сходимости
НаблюдаемостьАвтоматический анализ: если rank(Au) = p — все неизмеряемые потоки наблюдаемы; иначе — разделение на наблюдаемые и ненаблюдаемые

Автоматическое восстановление «испорченных» данных
Бизнес-процесс сведения балансов в ДПП.Балансы включает 19 шагов — от получения данных из внешних систем до экспорта согласованного баланса. Большинство шагов автоматизированы.
1
Получение данных из внешних систем
Автоматический сбор из БДРВ, ERP, ручного ввода. Формирование пакета данных для расчёта.
2
Конфигурирование производственной модели
Построение объектной модели производства на основе схемы предприятия.
3
Формирование первичной отчётности
Оперативный баланс, анализ выполнения плана, мнемосхемы технологического процесса.
4
Конфигурирование модели материального баланса
Графический редактор: создание и редактирование модели, настройка НСИ, параметров и атрибутов объектов.
5
Первичная фильтрация данных
Автоматическое отсеивание данных, выходящих за уставки, и некорректного ввода.
6
Анализ модели на полноту связей
Проверка графовой модели. При неполноте — возврат к редактированию модели или корректировке данных.
7
Поиск грубых ошибок
Автоматический поиск ГО в первичных данных (неисправность приборов, сбои передачи). Автоматическое исправление путём восстановления из имеющихся значений.
8
Согласование данных
Решение задачи сведения МБ с минимально возможными поправками. Все балансы должны сойтись — по производству и по отдельным участкам.
9
Анализ и формирование балансов
Анализ согласованных данных, выявление дебалансов, формирование итоговых балансов и балансовых отчётов.
10
Экспорт во внешние системы
Сохранение результатов расчёта МБ и экспорт в ERP, BI и другие информационные системы предприятия.
ДПП.Балансы — функциональное расширение цифровой платформы ДатаПлат. Архитектура соответствует рекомендациям Industrial Internet Consortium (IIC IIRA) и RAMI 4.0, а также ПНСТ РФ «Умное производство. Цифровые двойники» и «Промышленный интернет вещей».
