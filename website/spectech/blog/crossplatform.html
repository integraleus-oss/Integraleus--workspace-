Кроссплатформенность: Альфа на Linux и Windows

Обзор кроссплатформенных возможностей Альфа платформы. Рассматриваем поддержку российских дистрибутивов Linux, особенности развёртывания и требования безопасности.

Альфа платформа обеспечивает полноценную кроссплатформенную поддержку, работая как на Windows, так и на Linux. Это критически важно для российских предприятий, переходящих на отечественные операционные системы в соответствии с требованиями импортозамещения.

Особенности работы в Linux

Alpha.HMI и TCP/Alpha.Link

В Linux-системах Alpha.HMI использует протокол TCP/Alpha.Link для подключения к Alpha.Server вместо OPC DA. Это обеспечивает:

  • Удалённый доступ — HMI может находиться на отдельной машине
  • Кроссплатформенность — Windows HMI может подключаться к Linux серверу
  • Порты подключения — 4388 для оперативных данных, 4950 для истории
  • Безопасность — аутентификация с хэшированием паролей
# Конфигурация подключения Alpha.HMI к Linux-серверу
[AlphaLink]
ServerAddress = "192.168.1.100"
OperativePort = 4388
HistoryPort = 4950
Username = "operator"
PasswordHash = "sha256:..."
ConnectionTimeout = 30

Alpha.Historian в Linux

Архивный сервер распространяется в виде deb/rpm пакетов с полной интеграцией в Linux-систему:

# Установка и настройка Alpha.Historian в Linux
sudo dpkg -i alpha-historian_4.0_amd64.deb
sudo systemctl enable alpha.historian.server
sudo systemctl start alpha.historian.server

# Мониторинг статуса
sudo systemctl status alpha.historian.server
journalctl -u alpha.historian.server -f

Alpha.DevStudio на Linux

С 2025 года среда разработки Alpha.DevStudio полностью поддерживает Linux, обеспечивая:

  • Нативную производительность — без слоёв совместимости
  • Системную интеграцию — использование Linux-инструментов
  • CLI-автоматизацию — интеграция в CI/CD процессы
Миграция с Windows

Проекты, созданные в Windows, полностью совместимы с Linux. Требуется только пересборка конфигураций для изменения типов источников данных с OPC DA на TCP/Alpha.Link.

Безопасность и сертификация

Кроссплатформенная поддержка Альфа платформы включает соответствие требованиям информационной безопасности:

Сертификация безопасности

ФСТЭК России — соответствие требованиям по защите информации

КИИ категории 2-3 — применение на объектах критической информационной инфраструктуры

Российские дистрибутивы — сертифицированы для госсектора

Требования безопасности

  • Шифрование трафика — защита данных при передаче по TCP/Alpha.Link
  • Аутентификация — строгая проверка учётных данных
  • Аудит доступа — журналирование всех операций
  • Изоляция процессов — использование системных механизмов безопасности Linux

Преимущества кроссплатформенности

Гибкость развёртывания

  • Смешанные среды — Windows-операторы с Linux-серверами
  • Поэтапная миграция — постепенный переход на Linux
  • Выбор платформы — под конкретные требования проекта

Экономические выгоды

  • Снижение TCO — отсутствие лицензионных платежей за ОС
  • Импортозамещение — соответствие государственным требованиям
  • Независимость — снижение зависимости от зарубежных поставщиков

Технические преимущества

  • Стабильность Linux — высокое время безотказной работы серверов
  • Производительность — эффективное использование ресурсов
  • Масштабируемость — горизонтальное расширение в облаке
# Пример развёртывания в смешанной среде
# Linux-сервер (Alpha.Server + Alpha.Historian)
sudo systemctl start alpha.server
sudo systemctl start alpha.historian.server

# Windows-операторы (Alpha.HMI)
# Подключение через TCP/Alpha.Link к Linux-серверу
# IP: 10.0.1.100, Port: 4388
Рекомендации по развёртыванию

Серверные компоненты — Linux для высокой стабильности и безопасности. АРМ операторов — Windows или Linux в зависимости от предпочтений пользователей. Разработка — любая платформа с Alpha.DevStudio.

Кроссплатформенная поддержка Альфа платформы обеспечивает гибкость выбора технологического стека при сохранении полной функциональности и соответствии требованиям информационной безопасности.
