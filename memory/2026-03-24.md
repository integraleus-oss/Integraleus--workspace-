
## HumanLike Agent — Фаза 1: Достоверность поведения (06:01 MSK)

### Что сделано
Полная переработка `human_simulation.py` и адаптация `agent.py`:

1. **Адаптивная скорость печати** — 7-16 cps вместо константы 12. Быстрее в активном диалоге, медленнее ночью, зависит от длины текста.
2. **Typing-анимация с паузами** — `calculate_typing_chunks()` разбивает печать на этапы, иногда typing пропадает (человек задумался) и появляется снова. 25% шанс паузы в длинных текстах.
3. **Реалистичные опечатки** — 5 типов: соседняя клавиша (40%), перестановка букв (25%), пропуск буквы (15%), удвоение (10%), пропуск пробела (10%). Исправление через `*слово`.
4. **Engagement-адаптация** — per-chat трекинг состояния. В активном диалоге ответ за 0.5-3 сек, первый ответ после паузы — 3-90 сек ("открывает телегу").
5. **Имитация "открытия приложения"** — `simulate_app_opening()`: задержка зависит от того, как давно агент последний раз писал (5 сек если недавно, до 2 мин если давно).
6. **"Прочитал, но не ответил"** — 10% шанс пометить прочитанным, но ответить через 30-180 сек. С `ReadHistoryRequest`.
7. **Человечная разбивка** — по `\n\n`, `\n`, предложениям. 2-4 части вместо ровно 2. 40% шанс отправить целиком.
8. **Время суток** — фактор замедления: 1.0 днём, 1.5 вечером, 3.0 ночью.
9. **Отмена отложенных** — если пришло новое сообщение пока агент "думает" — предыдущий отложенный ответ отменяется.

## HumanLike Agent — Фаза 3: Адаптивное обучение (06:41 MSK)

### Что сделано
Модуль `chat_learner.py` — агент учится из каждого чата и адаптирует поведение.

1. **Анализ стиля сообщений**:
   - Формальность (0-1): маркеры формального/неформального стиля
   - Emoji-частота: считает эмодзи и скобочки-смайлы
   - Сленг-уровень: detects разговорные маркеры
   - Длина сообщений: средняя по чату
   - Стиль пунктуации: normal/ellipsis/emphatic/none

2. **Экспоненциальное скользящее среднее** — не равные веса, а EMA (α=0.3 для начала, α=0.1 после 20 сообщений). Быстро учится на новых чатах, стабилизируется потом.

3. **Per-chat + per-user профили** — SQLite таблицы chat_style и user_style. Знает стиль каждого пользователя в каждом чате.

4. **Адаптивный промпт** — `build_style_instructions()` добавляет инструкции:
   - Неформальный чат → "Сленг ок, скобочки, без заглавных"
   - Формальный → "Вежливые обороты, полные предложения"
   - Мало emoji → "НЕ используй эмодзи"
   - Короткие сообщения → "1-2 предложения максимум"

5. **Адаптивный max_tokens** — в коротком чате LLM получает 100 токенов, в длинном 500+

6. **Активные часы** — трекинг когда чат живёт (пиковые часы)

7. **Команда /style** — показывает выученный профиль чата (формальность, длина, emoji, пиковые часы, топ юзеры)

### Файлы
- `projects/humanlike-agent/chat_learner.py` — НОВЫЙ (12.6KB)
- `projects/humanlike-agent/agent.py` — интеграция learner
- `projects/humanlike-agent/commands.py` — +/style команда

## Alpha-Bot — Фаза 3: Аналитика (06:45 MSK)

### Что сделано
1. **Детекция намерений** — 7 типов:
   - `price_interest` — интерес к ценам/лицензированию
   - `contact_request` — оставляет email/телефон
   - `demo_request` — хочет скачать/попробовать
   - `comparison` — сравнивает с конкурентами
   - `technical_deep` — глубокие тех. вопросы
   - `followup` — переспрашивает (индикатор непонимания)
   - `satisfaction` — доволен ответом

2. **Воронка конверсии** — от "зашёл" до "оставил контакт":
   всего сессий → вовлечённых (>2 сообщ.) → интерес к цене → запрос контактов

3. **Качество ответов**:
   - Satisfaction rate — % довольных сессий
   - Followup rate — % переспросов (чем меньше, тем лучше)
   - Средняя длина диалога

4. **Топ тем** — какие модули/вопросы самые популярные

5. **Дневная статистика** — вопросы и сессии по дням с визуализацией

6. **Команда `/analytics [days]`** — полный отчёт в Telegram

7. **Web endpoint** — `GET /api/analytics?days=30` → JSON для внешнего дашборда
