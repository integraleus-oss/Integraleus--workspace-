
## Alpha licensing — session notes (group: Общий офис, topic 2)

Date: 2026-03-06

### What was fixed as mandatory process
- Before any Alpha calculation, preflight must be done against:
  1) playbooks/PRESELL_FASTLANE.md
  2) playbooks/EDGE_CASES.md
  3) data/licensing_automiq/rules_hard_checks.md
- Final calculation must always be validated by tariff/policy (2026 for current work).
- In answers: provide SKUs + total system price; do NOT provide line-item SKU prices unless explicitly requested.
- For Alpha.Platform, explicitly state that drivers are included and do not require separate purchase.

### Decision rules confirmed today
- If reserve is required -> baseline recommendation starts from Alpha.SCADA logic (with reserve accounted correctly).
- If no reserve and up to 50k tags with 1 client -> Alpha.One+ is acceptable to propose.
- For signal-based calculations: ask/collect signals first, then information power level.
- Exchange One+/SCADA -> Platform is treated as external by default (OPC UA), tags count as external on receiving side.
- Internal tags are not counted external only for Platform<->Platform via Alpha.Link.

### Commercial/format rules confirmed
- Always indicate tariff year in final output.
- Usually provide at least 2 options (SCADA + Platform), but tailor to architecture.
- For WEB > 5 concurrent users, explicitly mention recommendation/need for dedicated WEB server and WEB-PORTAL where applicable.
- For Full clients, clarify terminal/RDP split; if unknown, mark "to be уточняется при заказе ключей".

### Rule clarification (important)
- Pattern "common center + remote systems" is reusable for any number of remote systems (not only 29).
- The value "29" was just one concrete case parameter used in calculations.
- Local Historian requirement remains case-dependent and must be explicitly confirmed per case.
