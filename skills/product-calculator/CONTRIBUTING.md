# Contributing to Product Calculator

First off, thank you for considering contributing to the Universal Product Calculator! 🎉

## 🚀 Quick Start

1. **Fork** the repository
2. **Clone** your fork locally
3. **Create** a new branch for your feature
4. **Make** your changes
5. **Test** thoroughly
6. **Submit** a pull request

## 🛠️ Development Setup

```bash
# Clone your fork
git clone https://github.com/your-username/product-calculator-skill.git
cd product-calculator-skill

# Create a feature branch
git checkout -b feature/your-amazing-feature

# Make changes and test
# (Open calculator.html in your browser)

# Commit your changes
git add .
git commit -m "Add amazing feature that does X"

# Push to your fork
git push origin feature/your-amazing-feature
```

## 📝 Types of Contributions

### 🐛 Bug Reports
Found a bug? Please create an issue with:
- Clear description of the problem
- Steps to reproduce
- Expected vs actual behavior
- Browser/device information
- Screenshots if applicable

### ✨ Feature Requests
Have an idea? Open an issue with:
- Clear description of the feature
- Use case and benefits
- Possible implementation approach
- Examples from other calculators

### 🔧 Code Contributions
We welcome:
- Bug fixes
- New features
- Performance improvements
- Better documentation
- UI/UX enhancements
- Accessibility improvements

## 🎯 Development Guidelines

### Code Style
- Use **semantic HTML** elements
- Follow **BEM CSS** methodology for class names
- Write **clean, readable JavaScript**
- Use **modern ES6+** features
- Include **JSDoc comments** for functions

### Testing
Before submitting:
- ✅ Test on multiple browsers (Chrome, Firefox, Safari)
- ✅ Test on mobile devices
- ✅ Verify all calculations are correct
- ✅ Check theme switching works
- ✅ Validate form submissions
- ✅ Test edge cases (zero values, large numbers)

### Documentation
- Update README.md if needed
- Add examples for new features
- Document breaking changes
- Include screenshots for UI changes

## 🎨 UI/UX Guidelines

### Design Principles
- **Clean and minimal** — Don't overwhelm users
- **Responsive first** — Mobile is equally important
- **Accessible** — Follow WCAG guidelines
- **Fast** — Optimize for performance
- **Intuitive** — Users shouldn't need instructions

### Visual Style
- Use the established color palette
- Maintain consistent spacing (8px grid)
- Follow typography hierarchy
- Keep animations subtle and purposeful
- Support both light and dark themes

## 🔍 Review Process

1. **Automated checks** — CI/CD runs basic tests
2. **Code review** — Maintainers review for quality
3. **Manual testing** — Test on different devices
4. **Documentation review** — Ensure docs are updated
5. **Merge** — Once approved, changes are merged

## 📋 Commit Message Format

Use clear, descriptive commit messages:

```
type(scope): description

Examples:
feat(calculator): add currency conversion
fix(ui): resolve mobile layout issue  
docs(readme): update installation guide
style(css): improve dark theme colors
perf(js): optimize price calculation
```
