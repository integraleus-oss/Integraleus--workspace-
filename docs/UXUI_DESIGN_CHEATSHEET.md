## Глава 1: UX-основы

### UI vs UX
- **UI** — визуальная часть: что пользователь видит и с чем взаимодействует
- **UX** — весь опыт: от первого контакта с брендом до выполнения задачи
- UI — как выглядит кнопка. UX — зачем она тут и что происходит после нажатия

### Персоны (User Personas)
- Вымышленные, но реалистичные описания реальных пользователей
- **Компоненты**: фото, имя, био, бренды, поведение, цели, болевые точки, мотивация
- Не больше 1-2 персон — иначе теряется фокус
- Основаны на **исследованиях** (минимум 5 пользователей на тип)

### Исследования
- **Количественные**: A/B тесты, опросы, eye-tracking — что измеримо
- **Качественные**: интервью, дневники, наблюдение — почему так
- **Attitudinal** (что говорят) vs **Behavioral** (что делают)
- Всегда комбинировать оба типа

### User Flow
- Визуализация пути пользователя от входа до цели
- Три вопроса: **Кто** пользователь? **Какая** цель? **Какие** шаги?
- Помогает найти, где оптимизировать

### Вайрфреймы
- Скелет страницы — без цветов и шрифтов
- Сначала на бумаге, потом digital
- Три критерия: **ясность**, **уверенность пользователя**, **простота**
- Не привязывайтесь к решению — оно может провалить тест

## Глава 2: Визуальные принципы и Design Thinking

### Design Thinking — 6 шагов
1. **Empathize** — понять пользователя (интервью, наблюдение)
2. **Define** — определить проблему до генерации идей
3. **Ideate** — генерировать широкий спектр решений, сужать
4. **Prototype** — воплотить в первый тестируемый вид
5. **Test** — проверить с реальными пользователями
6. **Implement** — внедрить

### Эмпатия в дизайне
- Эмпатия ≠ сочувствие. Это **прожить** опыт пользователя
- При создании персон — представь себя на их месте
- При дизайне интерфейса — ставь эго в сторону, слушай фидбэк

### Инструменты
- **Figma** — браузерный, бесплатный, коллаборативный (основной)
- **Adobe XD** — лендинги, знакомым с Photoshop
- **Balsamiq** — вайрфреймы (ограничивает от лишних деталей)
- **Lucidchart** — флоучарты, user flows
- **Webflow** — от вайрфрейма до готового сайта

## Глава 3: Типографика

### Ключевые принципы
- **Иерархия**: размер, цвет, контраст, выравнивание → что читать первым
- **Единообразие**: один typeface (primary + secondary шрифт из него)
- **Читаемость**: x-height 60-75% от cap height, достаточный letter-spacing
- **Доступность**: контраст тела ≥ 4.5:1, крупный текст ≥ 3:1 (WCAG)

### Типы шрифтов
- **Serif** (засечки): Humanist → Old Style → Transitional → Didone → Slab
- **Sans Serif**: Grotesque → Neo-Grotesque → Humanist Sans → Geometric
- **Script**: Formal (каллиграфия) и Casual (кисть, маркер)

### Text vs Display typeface
- **Text**: для body copy — приоритет читаемости, открытые counters, ровный "цвет" текста
- **Display**: для заголовков — больше свободы для выразительности, но всё равно legible
- Тест: размой взгляд — если видны тёмные пятна, шрифт неровный

### Typesetting
- **Kerning** — межсимвольное расстояние
- **Tracking** — межсловное расстояние
- **Leading** — межстрочное расстояние

## Глава 4: Дизайн-системы и Figma

### Зачем дизайн-система
- **Единообразие** — единый источник правды для цветов, типографики, компонентов
- **Качество** — масштабируемость, профессиональный вид
- **Коммуникация** — дизайнеры + разработчики на одной странице

### Три части дизайн-системы
1. **Style Guide** — шрифты, цвета, tone of voice
2. **Pattern Library** — UI-паттерны (формы, кнопки), протестированные на юзабилити
3. **Component Library** — то же + код (экономия на разработке)

### Популярные системы для изучения
- **Material Design** (Google) — самая документированная
- **Human Interface Guidelines** (Apple)
- **Atlaskit** (Atlassian)
- **Lightning** (Salesforce) — CSS-фреймворк
- **Polaris** (Shopify)

## Глава 5: Психология цвета

### Элементы цвета
- **Hue** — базовый цвет
- **Value** — светлость/темнота
- **Saturation** — насыщенность
- **Chroma** — чистота
- **Tints** (+ белый), **Shades** (+ чёрный), **Tones** (+ серый)

### Правила цвета в UI
- **60-30-10**: доминирующий — вторичный — акцент
- **Branding color**: 1-2 основных
- **Functional**: навбар, текст, ссылки
- **Feedback**: зелёный=успех, красный=ошибка
- **Contrast**: обязателен для читаемости и доступности
- **Colorblind**: избегать пар красный-зелёный, синий-жёлтый

### Цветовые палитры
- Monochromatic, Analogous, Complementary, Split-Complementary, Triadic, Tetradic
- Проверка: обесцветить проект → элементы должны различаться в grayscale

### Золотое сечение (1.618)
- Типографика: если C=10pt, то B=10×1.618≈16pt, A≈26pt
- Изображения: Golden Spiral для фокальных точек
- Layout: самое важное — в центр спирали

## Глава 6: Сетки и модальные окна

### 12-колоночная сетка
- Индустриальный стандарт для веб
- Columns (одинаковые) + Gutters (промежутки) + Margins (поля)
- 12 делится на 2, 3, 4, 6 — гибко

### Когда ломать сетку
- Hero-изображения — на всю ширину
- Элементы с асимметрией для привлечения внимания
- CTA — чуть за пределами сетки
- Правило: бо́льшая часть контента в сетке, ломай **осознанно и редко**

### Modes & Modals
- **Mode** — состояние системы, меняющее поведение одного действия
- **Modal** — попап, блокирующий фон (пользователь обязан отреагировать)
- Modes — частый источник ошибок (пользователь не понимает, в каком состоянии система)
- Использовать модалки только для **критичных** действий (подтверждение удаления и т.д.)

## Применение

### Типографика
- [ ] Проверить контраст body text: ≥ 4.5:1 (WCAG)
- [ ] Один typeface family (Inter уже используется — ок)
- [ ] Иерархия: H1 > H2 > body чётко различима
- [ ] Достаточный line-height (leading)

### Цвет
- [ ] Правило 60-30-10 на главной
- [ ] Feedback colors: success/error в формах
- [ ] Grayscale-тест: всё различимо без цвета?

### Сетка
- [ ] Единая grid system по всем страницам
- [ ] Hero может ломать сетку, контент — нет

### Формы (калькулятор, контакт)
- [ ] User flow прописан: откуда пришёл → что заполняет → что получает
- [ ] Вайрфрейм-мышление: зачем каждый элемент на странице?

### Общее
- [ ] Персона инженера АСУ ТП: цели, боли, мотивация
- [ ] Design Thinking: эмпатия к пользователю в каждом решении
