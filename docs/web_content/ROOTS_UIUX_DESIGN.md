## ЧАСТЬ I — Сетка и Layout

### 12-колоночная сетка (Desktop)
- **Стандарт:** 12 колонок, гаттеры 12/14/16pt, маргины 160/180pt
- **Правило 8pt:** все размеры элементов кратны 8 (16, 24, 32, 40...)
- **Fluid Grid** — колонки адаптируются к ширине экрана, маргины и гаттеры фиксированы → для респонсивных интерфейсов
- **Fixed Grid** — ничего не меняется при изменении экрана → для фиксированного контента

### Мобильная сетка
- 8pt система, маргины 16-24px, гаттеры 16px (минимум)
- Baseline grid для текста — 4px шаг

## ЧАСТЬ II — Типографика

### Гарнитура vs Шрифт
- **Typeface (гарнитура)** = семейство (Open Sans, Roboto)
- **Font (шрифт)** = конкретный стиль (Regular, Bold, Italic)
- Выбирайте по **читаемости, масштабируемости, наличию весов**

### Serif / Sans Serif / Script
- **Sans Serif** — чистый, современный, для экранов
- **Serif** — элегантный, классический
- **Script** — избегать в UI (плохая читаемость)

### Интерлиньяж (Line Height)
- Маленький текст → множитель 1.6 (16px × 1.6 = 26px)
- Крупный текст → меньший множитель
- **Длина строки:** слишком короткая — рвёт поток; слишком длинная — глаза теряются

### Межбуквенный интервал (Letter Spacing)
- По умолчанию = 0, менять осторожно
- Для CAPS — увеличивать (kerning)
- Хорошие шрифты уже оптимизированы

### Выравнивание
- **90% людей — правши, большинство языков LTR → левое выравнивание по умолчанию**
- Центрирование — только для коротких текстов
- Следите за «рваным краем» (rag) справа

### Шкала типографики (Type Scale)
- 10+ размеров текста с иерархией
- **Не более 3 размеров шрифта в одной секции**
- Онлайн-инструменты: type-scale.com

## ЧАСТЬ III — Цвета

### Роль цвета
- Пользователь формирует мнение за **90 секунд**
- Цвет = настроение + тон + доверие

### Форматы
- **HEX** — веб (основной)
- **RGB** — экраны
- **CMYK** — печать
- **Pantone** — стандартизация брендов

### Почему НЕ чистый чёрный (#000)
- Слишком контрастен на белом → напряжение глаз
- Белый текст на чёрном → «halation» (буквы расплываются)
- **Используйте тёмно-серый** вместо чистого чёрного

### Палитра за 5 шагов:
1. **Primary color** — основной (кнопки, иконки, заголовки)
2. **Secondary color** — дополнительный (вторичные действия)
3. **Attention colors** — информация (синий), успех (зелёный), предупреждение (жёлтый), ошибка (красный)
4. **Tints & Shades** — вариации для фонов, карточек, оверлеев
5. **Золотое сечение 60-30-10:**
   - 60% — доминирующий (обычно нейтральный)
   - 30% — вторичный
   - 10% — акцент

### Контраст
- Проверяйте на contrast-ratio.com
- WCAG стандарты доступности

### Dark Mode
- Tints контрастнее, чем shades
- НЕ белые тени — используйте более тёмные оттенки + светлый tint фона

## ЧАСТЬ IV — Градиенты

- **Linear** — для квадратных/полигональных форм; свет сверху
- **Radial** — для круглых форм; 3D эффект
- **Angular (Conic)** — для особых эффектов
- Максимум 3 цвета в маленьких формах
- Между комплементарными цветами добавляйте промежуточный (иначе серость)
- **Вдохновение из природы** — самые естественные переходы

---

## ЧАСТЬ V — Тени

### Зачем тени
- Иерархия элементов
- Восприятие глубины
- Направление движения
- Определение границ

### Типы:
- **No Shadow** — элемент лежит на фоне
- **Drop Shadow** — элемент «парит»
- **Inner Shadow** — элемент «вдавлен» (Neumorphism, инпуты)

### Как делать реалистичные тени:
- **Две тени:** core shadow (тонкая, тёмная, близко) + cast shadow (широкая, светлая, дальше)
- Не ставьте тень сверху (свет идёт сверху → тень снизу)
- Разный размер тени для разных элементов
- **Dark Mode:** НЕ белые тени → тёмный оттенок элемента + светлый tint фона

## ЧАСТЬ VI — Кнопки

### Психология скруглённых углов
- Скруглённые → внимание внутрь, меньше когнитивная нагрузка
- Острые углы → внимание наружу

### Размеры
- Минимум touch target: **36×36px** (Material Design), **40×40px** (Apple HIG)
- Base button: текст 16px, padding 12px, ширина = 2× высота

### Иерархия кнопок
- **Primary** → главное действие (яркий, контрастный)
- **Secondary** → второстепенное (outline, серый)
- **Tertiary** → минимальное действие (текстовая ссылка)

### Состояния: Default → Hover → Active → Progress → Focus → Disabled

### Цвета кнопок
- Синий = информация
- Зелёный = успех
- Жёлтый = предупреждение
- Красный = опасность/ошибка

### Размещение
- **F-pattern и Z-pattern** — ставьте кнопки в зоны сканирования
- «Вперёд» = справа, «Назад» = слева
- Для деструктивных действий (удаление) — инвертируйте позиции

## ЧАСТЬ VII — Формы

### Инпуты
- Placeholder text — обязательно (примеры ввода)
- Скруглённые углы предпочтительнее
- Иконки в полях — повышают интуитивность
- **Состояния:** Default, Focus, Error, Success, Disabled
- Сообщения об ошибках → конкретные + как исправить
- Узкие инпуты для коротких данных (телефон, индекс)

### Dropdown
- Рекомендуется при > 4 вариантах
- Поиск внутри дропдауна — улучшает UX
- Scrollbar + визуальные категории / разделители

### Radio Buttons
- Для взаимоисключающих вариантов (выбор одного из N)

### Checkboxes
- Для множественного выбора
- Touch area > 48px на мобильных

### Toggle
- Два состояния (On/Off)
- НЕ писать ON/OFF в кнопке — различайте цветом

### Длинные формы → разбивать на шаги (wizard)

## ЧАСТЬ VIII — Иконки

### Функции:
- **Clarification** — пояснение текста
- **Interactive** — действие (кнопка)
- **Decorative** — эстетика

### Правила:
- **Простота** — сложная иконка = непонятная
- **Масштабируемость** — фрейм 16/24/32px
- **Консистентность** — один стиль, ≤ 3-4 цвета, единая толщина линий
- **Скруглённые иконки** — как и кнопки, приятнее глазу

### Бесплатные библиотеки:
heroicons.com, icons8.com, materialdesignicons.com, nucleoapp.com

## ЧАСТЬ IX — Изображения

### Ключевые правила:
- Мозг обрабатывает изображения в **60 000 раз быстрее** текста
- Если картинку сложно прочитать на маленьком экране → плохой выбор
- **Сжимайте** перед загрузкой (скорость + чёткость)
- **Визуальное единообразие** — mood board, единый стиль
- **Контекстная уместность** — картинка должна соответствовать теме

### Текст поверх изображения:
- Фильтр (чёрный/основной цвет), opacity 50-80%
- Обеспечивает читаемость

### Композиция:
- **«Правило третей»** — 9 частей, ключевые объекты на пересечениях линий
- Оставляйте «воздух» вокруг элементов

### Бесплатные фото: unsplash.com, pexels.com, pixabay.com

## ЧАСТЬ X — Навигация

### Типы:
- **Header Menu** — основные ссылки сверху
- **Sidebar Menu** — для админок/дашбордов
- **Footer Menu** — ресурсы, политики, соцсети, подписка
- **Breadcrumbs** — для сайтов > 2 страниц
- **Tabs (Desktop)** — переключение без перехода (цены, планы)
- **Tabs (Mobile)** — нижнее меню, макс. 5 вкладок

### Принципы:
- Не перегружайте навигацию
- Приоритизируйте главные страницы
- Простые взаимодействия, чистый дизайн

## ЧАСТЬ XI — Компоненты (Mastering UI/UX)

### Navbar
- Border radius > 8px (не острые углы)
- Иконки — не тем же цветом, что текст (opacity 50-60%)
- Gaps между ссылками ≥ 16px
- Высота ≤ 80px
- Стили: Simple (прозрачный), Light, Dark, Blurred

### Hero Section
- Заголовок ≤ 10 слов, H1
- Подзаголовок H5/H6
- Не перегружать элементами
- **Показывайте часть следующей секции** → побуждает скроллить
- Стили: Background Image, Side Image, Carousel, Background Video, Blur Effect

### Features Section
- Центрирование — для описаний ≤ 2 строки
- Левое выравнивание — для длинных описаний
- Не используйте полную ширину контентной области
- Для коротких фич (4-5 слов) — ✓ чекмарки вместо буллетов

### Cards
- Визуальная иерархия — важное наверху
- Контент лаконичный
- Одна ссылка на карточку
- Кнопки — различимые primary/secondary
- Изображения — одинаковые маргины, единый aspect ratio

### Таблицы (Tables)
- Шапка — фон + жирный текст
- Чередование строк (zebra striping)
- Горизонтальный скролл на мобильных → избегать
- Sticky header для длинных таблиц

### FAQs
- Accordion — самый распространённый
- Разбивайте по категориям при > 10 вопросах
- Одна тема = один FAQ раздел

### Contact Us
- Поля: имя, email, сообщение (минимум)
- Кнопка отправки — заметная
- Карта + адрес + телефон + часы работы

### Footers
- Ссылки, соцсети, копирайт, подписка
- Dark footer → если dark header
- Минимум текста, максимум ссылок

## ЧАСТЬ XII — Mobile Design

### Ключевые принципы:
- **Thumb-friendly:** touch targets ≥ 44×44px, spacing 8-10px
- **Навигация снизу** (thumb zone)
- **Читаемость** — крупнее текст на маленьких экранах
- **Упрощённые формы** — минимум ввода
- **Консистентность web ↔ mobile** — одинаковый UX, адаптированный layout

## ЧАСТЬ XIII — Структуры типовых страниц

### About Us
Hero → Logo Area (партнёры) → Features → Team → Stats → Projects → Footer

### Profile Page
Hero (аватар, имя, био) → Content (работы) → Reviews → Contact Form → Footer (соцсети)

### Blog Page
Hero (навигация, карусель) → Featured Posts → Post List (пагинация) → Other Articles → Footer (подписка)

### Blog Post
Hero (заголовок, дата, автор) → Content (иерархия типографики, line-height) → Author Bio → Related Articles → Prefooter (подписка) → Footer

### Product Page
Hero (фото, название, цена, наличие) → Features (доставка, возврат, теги) → Client Reviews → Other Products → Prefooter (подписка) → Footer

### Rental Page
Hero (фон + поиск) → Places (карточки) → Reviews → FAQ → Contact Form → Footer

### Application Page
Hero (CTA) → Features → Integrations → Pricing (планы рядом) → Testimonials → Footer

### Sign Up / Sign In
- Две опции: Classic (email/password) + Social Media (OAuth)
- Remember Me checkbox
- Ссылка на регистрацию/вход
- Валидация в реальном времени

### Legal Agreements
- Боковое меню с anchor links
- Чистая типографика
- Динамическое выделение текущего раздела

### Shopping Cart / Checkout
- Список товаров с фото, ценой, количеством
- Промокод
- Итого
- Checkout: shipping → payment → confirmation (wizard)

## UI/UX Тренды (по книге)

| Тренд | Характеристика |
|---|---|
| **Flat Design** | Чистые 2D элементы, яркие цвета, без теней |
| **Material Design** | Elevation, тени, Roboto, card-based layouts |
| **Neubrutalism** | Толстые рамки, крупная типографика, асимметрия |
| **Bauhaus** | Геометрия, primary colors, grid-системы |
| **Neumorphism** | Мягкие тени, элементы «вдавлены/выдавлены» из фона |
| **Glassmorphism** | Frosted glass эффект, прозрачность, яркие фоны |

## Быстрый чеклист UI/UX

- [ ] Сетка 12 колонок, размеры кратны 8pt
- [ ] Типографика: sans-serif, ≤ 3 размера в секции, line-height × 1.6 для body
- [ ] Цвета: палитра по 60-30-10, контраст проверен
- [ ] Не чистый чёрный (#000) — тёмно-серый
- [ ] Кнопки: скруглённые, ≥ 36px touch target, иерархия primary/secondary
- [ ] Тени: две (core + cast), не сверху
- [ ] Формы: placeholder, валидация, конкретные ошибки
- [ ] Иконки: один стиль, простые, масштабируемые
- [ ] Изображения: сжаты, единый стиль, фильтр под текст
- [ ] Навигация: не перегружена, ≤ 5 мобильных табов
- [ ] Hero: ≤ 10 слов заголовок, виден следующий раздел
- [ ] Mobile: thumb-friendly, крупный текст, упрощённые формы
