# Letting Go of the Words — Конспект для работы с сайтом

**Источник:** Janice (Ginny) Redish, "Letting Go of the Words: Writing Web Content That Works", 2nd Ed., 2012, Morgan Kaufmann.

---

## Главный принцип: Контент = Разговор

Каждое посещение сайта — это разговор, который начинает посетитель. Не вы начинаете — он. Ваша задача: поддержать этот разговор, ответить на вопросы, дать уйти с ответом.

**Веб = телефон, а не картотека.** Люди приходят не за документами, а за ответами.

---

## 1. Как люди читают в вебе

- **Сканируют, а не читают.** Большинство уходит с главной за 30 секунд.
- **F-паттерн:** читают первые строки, потом глаз скользит по началу строк вниз.
- **Читают, когда нашли то, что искали** — ответ на вопрос, нужную информацию.
- **Незрячие пользователи** сканируют «ушами» — перескакивают по заголовкам и ссылкам.

**Почему не читают больше:**
- Слишком заняты
- Контент нерелевантен их задаче
- Перегрузка информацией

---

## 2. Планирование: Зачем? Для кого? Какой разговор?

Перед написанием любого контента — три вопроса:

### Зачем? (Цели)
- Формулируйте через действия посетителя: не «мы хотим рассказать о продукте», а «мы хотим, чтобы люди купили / записались / нашли ответ»
- Будьте конкретны: «Я пишу это, чтобы (кто?) мог (сделать что?)»

### Для кого? (Персоны)
- Создайте персоны — конкретных людей с именем, фото, демографией, цитатами
- Персоны основаны на данных, а не догадках
- **Мы — не наши пользователи!**
- Учитывайте: опыт, эмоции, ценности, технологии, культурный контекст

### Какой разговор?
- Какие вопросы задаст эта персона?
- Какими словами она их сформулирует?
- Используйте слова посетителей, не переводите в свой жаргон

---

## 3. Дизайн для лёгкого использования

- **Интегрируйте контент и дизайн с самого начала** — не пишите текст отдельно
- **Используйте реальный контент в макетах** (не Lorem ipsum)
- **Пространство:** не бойтесь белого пространства; оно помогает читать
- **Типографика:**
  - Sans-serif для экрана
  - Не используйте ALL CAPS — труднее читать
  - Минимум курсива — плохо выделяется на экране
  - Размер текста ≥ 12px для основного
- **Цвет:**
  - Синий — только для ссылок
  - Проверяйте контрастность
  - Помните о дальтониках (~8% мужчин)
- **Не центрируйте текст** — левое выравнивание всегда лучше для чтения
- **Избегайте «ложного дна»** — не создавайте иллюзию, что страница закончилась

---

## 4. Главная страница

- Должна быть **найдена поисковиками** (SEO)
- **Идентифицирует сайт** — кто вы, чем занимаетесь
- **Задаёт тон и личность** сайта
- **Помогает быстро понять**, что здесь есть
- **Продолжает разговор** — быстро направляет дальше
- **Отправляет каждого по правильному пути**
- Контент-насыщенная, но с минимумом слов

---

## 5. Навигация и промежуточные страницы

- Люди **сначала охотятся**, потом читают
- **Не хотят читать** во время навигации
- Промежуточная страница = оглавление
- **«Три клика» — миф.** Важно не количество кликов, а ощущение прогресса
- Многие выбирают **первый вариант** — ставьте важное первым

---

## 6. Организация контента

- **Думайте «информация», а не «документ»**
- Делите контент по:
  - Вопросам, которые задают люди
  - Темам или задачам
  - Типу продукта / информации
  - Людям / аудитории
  - Жизненным событиям
  - Времени / последовательности
- **Каждая единица информации = карточка** (index card model)
- **Используйте PDF только когда это действительно нужно** (для печати, официальных форм)

---

## 7. Фокус на ключевых сообщениях ⭐

### 7 правил:

**1. Давайте только то, что нужно.**
- Процесс: отложите текущий текст → составьте список вопросов персоны → ответьте на каждый → отбросьте остальное

**2. Режьте! Режьте! И ещё режьте!**
- Фокус на фактах, убирайте «жир»
- Убирайте слова о себе / компании, если посетителю это не нужно
- Отложите на день, вернитесь, порежьте ещё
- Прочитайте вслух

**3. «Кусочек, перекус, обед» (Bite, Snack, Meal)**
- **Bite** = заголовок / ссылка + краткое описание
- **Snack** = ключевое сообщение / краткая суть
- **Meal** = полные детали
- Удовлетворяет все уровни потребности в информации

**4. Начинайте с ключевого сообщения.**
- Перевёрнутая пирамида: вывод — сначала
- Люди читают первые 1-2 абзаца, потом сканируют

**5. Используйте слои (layering).**
- Оверлеи для доп. информации (не уходя со страницы)
- Прогрессивное раскрытие (Show/Hide) — доступно для скринридеров

**6. Разрушайте стены текста.**
- Стены текста = барьеры
- Короткие абзацы или буллеты работают лучше всего
- Каждый раздел — свой заголовок
- **Кейс:** замена параграфа на буллет-лист увеличила конверсию с 5% до 7.3%

**7. Планируйте для шаринга в соцсетях.**
- Bite идёт в соцсеть → ведёт на Snack на сайте → Meal для деталей

## 8. Заголовки (Headlines)

### 7 правил:
1. **Используйте слова посетителей** — не ваш жаргон
2. **Ясность > креативность** — милые каламбуры не работают
3. **Думайте о глобальной аудитории** — метафоры могут не перевестись
4. **Средняя длина (~8 слов)** — не слишком коротко, не слишком длинно
5. **Утверждение, вопрос или призыв к действию** — не просто существительное
6. **Комбинируйте метки с информацией** — не просто «Доставка», а «Бесплатная доставка от 3000 ₽»
7. **Добавляйте короткое описание**, если людям нужен контекст

---

## 9. Подзаголовки (Headings)

### 11 правил:
1. **Не втыкайте заголовки в старый текст** — переосмыслите
2. **Начните с плана-скелета**
3. **Выберите стиль:** вопросы, утверждения, глагольные фразы
4. **Существительные — минимально** (они не продвигают разговор)
5. **Слова посетителей** в заголовках
6. **Параллелизм** — одинаковая структура на одном уровне
7. **Мало уровней** — 2-3 достаточно
8. **Заголовки визуально отличаются от текста**
9. **Каждый уровень — чётко различим** (размер, отступ, жирность)
10. **Якорные ссылки вверху** — для навигации внутри длинной страницы
11. **Проверка:** прочитайте только заголовки — складывается ли история?

**Правила дизайна заголовков:**
- Не синим (синий = ссылка)
- Не подчёркивать (если не ссылка)
- Не центрировать
- Линии — НАД заголовком, не под ним
- Заголовок ближе к своему тексту, чем к предыдущему

---

## 10. Предложения — 10 правил

1. **Обращайтесь к посетителю — «вы»**
   - Императив для инструкций (Нажмите, Выберите, Откройте)
   - «Вы» делает текст личным и понятным
2. **Используйте «я» и «мы»**
   - Вопрос = голос посетителя («Как я могу...?»)
   - Ответ = голос сайта («Вы можете...»)
   - Организация = «мы»
3. **Активный залог** (большей частью)
   - Пассив = скучно, непонятно, рискованно
   - Люди мысленно переводят пассив в актив — зачем заставлять?
4. **Короткие предложения (10-20 слов)**
   - Одна мысль = одно предложение
   - Фрагменты допустимы, если стиль позволяет
5. **Убирайте лишние слова**
6. **Допинформация — в своё место** (не вставляйте вводные в середину)
7. **Короткие абзацы** — 2-4 предложения, максимум
8. **Контекст — сначала** (given-new: сначала знакомое, потом новое)
9. **Действие — в глаголе** (не «осуществление проверки», а «проверьте»)
10. **Слова посетителей** — не ваш профессиональный жаргон

**Важно:** Ясный разговорный стиль ≠ «упрощение». Это уважение ко времени читателя.

## 11. Списки и таблицы

### Списки — 6 правил:
1. **Буллеты для вариантов/перечислений**
2. **Буллеты = стилю сайта** (кружки, квадраты, стрелки)
3. **Нумерация для инструкций** (пошагово)
4. **Короткие** — 5-10 пунктов для незнакомых тем
5. **Начинайте пункты одинаково** (параллельная структура)
6. **Форматируйте правильно** — отступы, пробелы между длинными пунктами

### Таблицы — 6 правил:
1. **Для «если → то»** ситуаций
2. **Для сравнения чисел**
3. **Таблица = ответ на вопрос**
4. **Первый столбец — тщательно** (по нему ищут)
5. **Простые** — 2-3 столбца, разбивайте большие
6. **Форматирование:**
   - Меньше линий, больше шейдинга
   - Текст — по левому краю (не по центру!)
   - Числа — по десятичной точке

---

## 12. Ссылки — 7 правил

1. **Не делайте незнакомые названия ссылками** — сначала объясните
2. **«Запуск = приземление»** — текст ссылки = заголовок целевой страницы
3. **Для действий — начинайте с глагола** (Получить, Скачать, Узнать)
4. **Осмысленные ссылки** — НЕ «Нажмите здесь», НЕ просто «Подробнее»
   - Скажите, подробнее *о чём*
   - «Подробнее о доставке», а не «Подробнее»
5. **Не встраивайте ссылки в текст** (для информационного контента) — ставьте после
6. **Буллеты рядом со ссылками — тоже кликабельные**
7. **Посещённые ссылки — другой цвет**
   - Цвет ссылок — только для ссылок
   - Не дразните цветом не-ссылки

---

## 13. Иллюстрации — 7 правил

1. **Не заставляйте гадать** — что на картинке и зачем она
2. **Подходящий размер** — не слишком маленький, не слишком большой
3. **Показывайте разнообразие** — разные люди, если показываете людей
4. **Контент ≠ реклама** — не делайте так, чтобы полезное выглядело как баннер
5. **Не раздражайте** — мигающий, бегущий текст → закрытие страницы
6. **Анимация — только если помогает** (демонстрация процесса)
7. **Доступность** — alt-текст, подписи к видео

**5 целей иллюстраций:**
- Точный товар (e-commerce)
- Помощь в самообслуживании (скриншоты, схемы)
- Показ процессов (диаграммы, видео)
- Графики и данные
- Создание настроения

## 14. Маркетинг на сайте

**Веб = pull, не push.** Посетитель приходит сам, со своей целью.

### Правила:
- **Сначала — ответьте на вопрос посетителя**, потом маркетинг
- **Не гоните маркетинг раньше времени** — это отталкивает
- **«Маркетинговый момент»** = момент, когда посетитель готов услышать предложение
  - После покупки → cross-sell
  - После ответа на вопрос → «Вам также может быть интересно»
- **Присоединяйтесь к разговору посетителя**, не меняйте тему

---

## 15. Юридические тексты

- **Точность + достаточность + ясность** — можно иметь все три
- Избегайте архаизмов («настоящим», «вышеуказанный»)
- Избегайте жаргона
- Заголовки словами пользователя
- Все правила ясного письма применимы к юридическим текстам

---

## 16. Стайл-гайд (органический)

- Начните с малого, наращивайте
- Каждая тема = маленькая карточка (не книга)
- Назначьте владельца
- Поддержка руководства
- Разрешайте разные стили для разных медиа (соцсети ≠ основной сайт)
- Тестируйте сам стайл-гайд

---

## 17. Тестирование

**Мантра: Test! Test! Test!**

- Не нужно: готового продукта, лаборатории, спец. ПО, видеозаписи
- Нужно: реальные люди, реальные задачи, реальные данные
- **5-15 участников**, по одному, ~1 час
- **«Утро в месяц»** — 3 человека, 1 утро, каждый месяц
- **A/B тесты** — количественные данные, но не объясняют «почему»
- **Фокус-группа ≠ юзабилити-тест** (говорят ≠ делают)
- **Тестируйте контент, а не только навигацию!**
  - Нашёл ли человек информацию?
  - Понял ли он её?
  - Может ли пересказать ключевое сообщение?

## Чеклист для веб-контента

- [ ] Определены цели страницы (что посетитель должен сделать?)
- [ ] Определена персона (для кого пишем?)
- [ ] Список вопросов персоны составлен
- [ ] Вопросы в порядке, логичном для посетителя
- [ ] Ключевое сообщение — в первом абзаце
- [ ] Заголовок ясный, со словами посетителя
- [ ] Подзаголовки рассказывают историю (читай только их — понятно?)
- [ ] Абзацы ≤ 4 предложений
- [ ] Предложения 10-20 слов
- [ ] Активный залог
- [ ] «Вы» обращение к посетителю
- [ ] Списки вместо длинных перечислений в абзацах
- [ ] Ссылки осмысленные (не «подробнее», не «нажмите здесь»)
- [ ] Запуск = приземление (текст ссылки = заголовок страницы)
- [ ] Иллюстрации с подписями и alt-текстом
- [ ] Нет стен текста
- [ ] Bite/Snack/Meal реализован
- [ ] Контент протестирован на реальных людях

## SEO через хороший контент

- Пишите для людей, не для поисковика (но помните о нём)
- **Ключевые слова** = слова, которыми люди ищут
- Ключевые слова — в заголовках, первых абзацах, alt-текстах, URL
- **Лучший SEO** = контент, который люди хотят, понимают и которым делятся
- Осмысленные ссылки помогают SEO
- «Запуск = приземление» помогает SEO
