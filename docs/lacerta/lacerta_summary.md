## Назначение
Проектирование, разработка и эксплуатация распределённых и локальных АСУ/РСУ.
Облачная архитектура, edge и fog computing.
Верхний уровень для АСУ ТП, АСУ ПП, IIoT.

## Области применения
- АСУ ТП, АСДУ/АСТУ
- РСУ/SCADA
- IIoT
- Диспетчерские центры
- Пункты управления энергоблоков
- ЦОД

## Состав платформы (6 компонентов)
1. **SCADA/HMI Runtime** — среда исполнения, мнемосхемы, виджеты, тренды, тревоги
2. **Промышленная СУБД** — СУБД реального времени, кэш РВ, архив, циклический архив (PostgreSQL 11 + ClickHouse 23)
3. **Шлюз сопряжения** — драйверы протоколов
4. **SCADA Designtime** — среда разработки проектов, поддержка SVG, синхронизация
5. **Библиотека элементов** — типовые элементы, SVG, импорт/экспорт
6. **Шлюз безопасности** — LDAP, Keycloak, межсетевые экраны, однонаправленные шлюзы

## Архитектура
- Кластерная модель 1+1 master-master
- Модули: lacerta-server (порт 8000), nginx (порт 80), PostgreSQL (5432), ClickHouse (9000)
- Web UI через браузер
- Событийная модель с тревогами
- Объектно-ориентированная модель данных
- Вычислительная модель: калькулятор, имитатор
- RBAC (ролевая модель доступа)
