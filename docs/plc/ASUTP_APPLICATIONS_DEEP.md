## 1. Типовая архитектура АСУ ТП

### Трёхуровневая модель

```
┌──────────────────────────────────────────────────────┐
│  Уровень 3: ИНФОРМАЦИОННО-ВЫЧИСЛИТЕЛЬНЫЙ            │
│  (корпоративный / MES / ERP)                         │
│                                                      │
│  Серверы: историк, MES, отчёты, бизнес-аналитика     │
│  Протоколы: OPC UA, SQL, REST API                    │
│  Сеть: корпоративная Ethernet (1 Гбит)               │
├──────────────────────────────────────────────────────┤
│  Уровень 2: ОПЕРАТИВНОГО УПРАВЛЕНИЯ                  │
│  (диспетчерский / SCADA)                             │
│                                                      │
│  АРМ оператора, АРМ инженера, серверы SCADA           │
│  Инженерные станции, серверы OPC, историки             │
│  Протоколы: OPC UA/DA, Modbus TCP, МЭК 104           │
│  Сеть: промышленная Ethernet (100 Мбит, резерв.)     │
├──────────────────────────────────────────────────────┤
│  Уровень 1: КОНТРОЛЛЕРНЫЙ (управления)               │
│                                                      │
│  ПЛК / контроллеры, модули ввода-вывода               │
│  Резервирование (hot standby), ПАЗ                    │
│  Протоколы: Profibus, Modbus RTU, HART, EtherCAT     │
│  Сеть: полевая шина / промышленный Ethernet           │
├──────────────────────────────────────────────────────┤
│  Уровень 0: ПОЛЕВОЙ                                  │
│                                                      │
│  Датчики: температура, давление, уровень, расход      │
│  Исполнительные механизмы: клапаны, задвижки,         │
│  приводы, пускатели                                   │
│  Сигналы: 4–20 мА, 0–10 В, дискретные 24 В DC        │
└──────────────────────────────────────────────────────┘
```

### Детализация уровней

**Уровень 0 — Полевой:**
- Датчики: RTD (Pt100), термопары, давление (4–20 мА), расходомеры (вихревые, электромагнитные, Кориолисовы), уровнемеры (радарные, ультразвуковые, ёмкостные)
- Исполнительные механизмы: регулирующие клапаны, шаровые краны с электроприводом, частотно-регулируемые приводы (ЧРП)
- Соединение: кабель 4–20 мА, HART, Foundation Fieldbus (редко)

**Уровень 1 — Контроллерный:**
- ПЛК выполняют алгоритмы управления, регулирования, защиты
- Модули I/O (DI, DO, AI, AO) — связь с полевым уровнем
- Контроллеры ПАЗ (SIS) — отдельные от основной АСУ ТП
- Связь с уровнем 2: Ethernet (Modbus TCP, OPC UA)

**Уровень 2 — Диспетчерский:**
- SCADA-система: визуализация, управление, архивирование
- АРМ оператора: мнемосхемы, тренды, аварийная сигнализация
- Историк: архив параметров (SOE, тренды, отчёты)
- Инженерная станция: конфигурирование, программирование ПЛК

**Уровень 3 — Информационный (опционально):**
- MES (Manufacturing Execution System): управление производством
- Связь с ERP (SAP, 1C и др.)
- Отчётность, KPI, энергоменеджмент

### Требования к сетям

| Сеть | Требования |
|------|-----------|
| Полевая шина (Уровень 0-1) | Детерминизм, помехозащищённость, длина до 1200 м |
| Контроллерная (Уровень 1-2) | Резервирование (кольцо, RSTP), QoS, 100 Мбит+ |
| Информационная (Уровень 2-3) | Безопасность (firewall, DMZ), 1 Гбит+ |

## 2. DCS vs PLC — сравнение

### Определения
- **PLC (ПЛК)** — программируемый логический контроллер. Автономное устройство для управления дискретными и непрерывными процессами
- **DCS (РСУ)** — распределённая система управления. Интегрированная платформа: контроллеры + I/O + SCADA + историк + инжиниринг

### Сравнительная таблица

| Критерий | PLC-based система | DCS |
|----------|-------------------|-----|
| **Архитектура** | ПЛК + отдельная SCADA + отдельный историк | Единая интегрированная платформа |
| **Масштабируемость** | Сотни — тысячи I/O | Тысячи — десятки тысяч I/O |
| **Стоимость (малые)** | Ниже | Выше |
| **Стоимость (большие)** | Может быть выше (интеграция) | Ниже (всё включено) |
| **Время цикла** | 1–10 мс | 100–500 мс (типично) |
| **Резервирование** | Опционально (дополнительные модули) | Встроенное (CPU, I/O, сеть) |
| **Интеграция** | Необходима интеграция компонентов | «Из коробки» |
| **Vendor lock-in** | Можно комбинировать вендоров | Сильная привязка к вендору |
| **Дискретное управление** | Отлично | Среднее |
| **Непрерывное управление** | Хорошо (с настройкой) | Отлично |
| **Сложное регулирование** | Требует разработки | Встроенные блоки (advanced PID, MPC) |
| **Batch-управление** | Требует интеграции | Встроенные модули ISA-88 |
| **Модификация онлайн** | Зависит от среды | Стандартная возможность |

### Когда выбирать PLC
- Малые и средние объекты (до 2000–5000 I/O)
- Преобладание дискретного управления (конвейеры, упаковка, дозирование)
- Быстрые процессы (Motion Control, сервоприводы)
- Бюджетные проекты
- Простые процессы с минимумом регулирования
- Необходимость нестандартных протоколов

### Когда выбирать DCS
- Крупные непрерывные производства (нефтепереработка, химия, энергетика)
- Большое количество контуров регулирования (ПИД, каскадное, MPC)
- Высокие требования к надёжности и резервированию
- Batch-процессы по ISA-88
- Интеграция с системами безопасности (SIS)
- Долгосрочная эксплуатация с модернизацией

### Тенденция: сближение
Современные PLC-системы приближаются к DCS (CODESYS + WebVisu + OPC UA), а DCS становятся доступнее. Гибридные решения (PLC+SCADA единой платформы) занимают нишу «малых DCS».

## 3. Типовые задачи автоматизации

### 3.1 ПИД-регулирование

**Принцип:**
```
Уставка (SP) ──┐
                ├── Ошибка (e) → ПИД-регулятор → Управляющее воздействие (OUT)
Процесс (PV) ──┘                                         │
     ↑                                                    ↓
     └──────────── Объект управления ◄────────────────────┘
```

**Формула ПИД:**
```
OUT = Kp × e + Ki × ∫e·dt + Kd × de/dt
```

**Настройка (метод Циглера-Никольса):**
1. Выключить интегральную и дифференциальную составляющие (Ki=0, Kd=0)
2. Увеличивать Kp до появления автоколебаний → критическое усиление Ku
3. Замерить период колебаний Tu
4. Рассчитать: Kp=0.6×Ku, Ti=Tu/2, Td=Tu/8

**Пример в ST:**
```iec-st
(* Регулирование температуры теплообменника *)
fbPID(
    rPV     := rTempOut,          (* факт. температура на выходе *)
    rSP     := rTempSetpoint,     (* уставка 65°C *)
    rKp     := 2.5,
    rKi     := 0.1,
    rKd     := 0.5,
    rOutMin := 0.0,
    rOutMax := 100.0
);
rValvePosition := fbPID.rOutput;   (* управление клапаном теплоносителя *)
```

### 3.2 Каскадное регулирование

**Применение:** когда объект имеет промежуточную (быструю) и конечную (медленную) переменные.

```
                    ┌──────────┐          ┌──────────┐
SP_внешний → [ПИД внешний] → SP_внутренний → [ПИД внутренний] → OUT → Клапан
                    ↑                      ↑
                    PV_внешний              PV_внутренний
                    (температура)           (расход)
```

**Пример:** Регулирование температуры колонны подачей орошения:
- Внешний контур: температура верха колонны → задание расхода орошения
- Внутренний контур: расход орошения → положение клапана

### 3.3 Противоаварийная защита (ПАЗ)

**Типовая логика:**

```iec-st
(* Аварийный останов компрессора *)
bTrip := (rPressure > rPressureTrip)           (* высокое давление *)
      OR (rTempBearing > rTempBearingTrip)     (* перегрев подшипника *)
      OR (rVibration > rVibrationTrip)         (* высокая вибрация *)
      OR (NOT bOilPressOK)                     (* низкое давление масла *)
      OR bEmergencyButton;                     (* аварийная кнопка *)

IF bTrip THEN
    bCompressorRun := FALSE;     (* останов компрессора *)
    bBlowdownValve := TRUE;      (* открыть свечу разгрузки *)
    bAlarmHorn := TRUE;          (* звуковой сигнал *)
END_IF
```

### 3.4 Телемеханика

**Задача:** передача данных с удалённых объектов (КТП, насосные, подстанции) на диспетчерский пункт.

**Типовая архитектура:**
```
┌─────────────┐     МЭК 104     ┌──────────────┐
│ Контролируе-│ ────TCP/IP────→ │ Диспетчерский │
│ мый пункт   │     или         │ пункт         │
│ (КП)        │  МЭК 101 (RS)  │ (ДП)          │
│             │                 │               │
│ ПЛК + I/O  │                 │ SCADA +       │
│ (TREI-5B,   │                 │ Историк +     │
│  ЭЛСИ-ТМК)  │                 │ ОИК           │
└─────────────┘                 └──────────────┘
```

**Передаваемые данные:**
- ТИ (телеизмерения): напряжение, ток, мощность, давление, температура
- ТС (телесигнализация): положение коммутационных аппаратов, сигнализация
- ТУ (телеуправление): включение/отключение, изменение уставок

## 4. Примеры проектов по отраслям

### 4.1 Нефтегаз

**Типовой объект:** ДНС (дожимная насосная станция), УКПН (установка комплексной подготовки нефти)

**Состав АСУ ТП:**
- Основная система управления: ПЛК (REGUL R500, ОВЕН ПЛК210) + SCADA
- Система ПАЗ: сертифицированный контроллер (HIMA, Yokogawa ProSafe)
- Пожарная сигнализация и газодетекция (F&G)
- Телемеханика: связь с диспетчерской через МЭК 104

**Типовые задачи:**
- Регулирование уровня в сепараторах (ПИД + каскад)
- Управление насосными агрегатами (пуск/стоп, АВР, наработка)
- Управление печами нагрева нефти (многозонное регулирование)
- Коммерческий учёт нефти и газа (расходомеры, вычислители)
- Блокировки при превышении параметров (давление, уровень, температура)

### 4.2 Энергетика

**Типовой объект:** электрическая подстанция 110/35/6 кВ

**Состав АСУ ТП:**
- Контроллеры присоединений (терминалы РЗА)
- Контроллер общеподстанционного уровня
- АРМ оператора (SCADA)
- Связь с диспетчерским центром: МЭК 60870-5-104
- Для «цифровых» подстанций: МЭК 61850 (GOOSE, MMS, SV)

**Типовые задачи:**
- Телеизмерение: напряжение, ток, мощность (активная, реактивная)
- Телесигнализация: положение выключателей, разъединителей
- Телеуправление: включение/отключение выключателей
- Регулирование напряжения (РПН трансформатора)
- Учёт электроэнергии (АСКУЭ)

### 4.3 Водоснабжение и водоотведение

**Типовой объект:** станция водоподготовки, насосная станция 2-го подъёма

**Состав АСУ ТП:**
- ПЛК (ОВЕН ПЛК210 / аналог) + SCADA
- ЧРП для насосов (управление по давлению)
- Модули ввода-вывода (уровнемеры, расходомеры, датчики давления)

**Типовые задачи:**
- Поддержание давления в водопроводной сети (ПИД + ЧРП)
- Каскадное управление насосами (подключение/отключение по давлению)
- Контроль уровня в резервуарах чистой воды
- Дозирование реагентов (хлор, коагулянт)
- Диспетчеризация через МЭК 104 или Modbus TCP

### 4.4 Металлургия

**Типовой объект:** прокатный стан, доменная печь, конвертер

**Состав АСУ ТП:**
- Быстрые контроллеры (время цикла < 1 мс) для Motion Control
- DCS для технологического управления
- Система ПАЗ
- Уровень 2: математические модели процесса

**Типовые задачи:**
- Управление приводами прокатного стана (синхронизация, позиционирование)
- Регулирование температуры в печах (многозонное, с адаптацией)
- Управление конвертерной плавкой (шихтовка, продувка, температура)
- Весовое дозирование шихтовых материалов
- Система MES: отслеживание продукции (tracking)
