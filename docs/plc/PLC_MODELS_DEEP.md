## 1. ОВЕН ПЛК210

### Общие сведения
- **Производитель:** ОВЕН (Россия, Москва)
- **Класс:** моноблочный контроллер для средних и распределённых систем автоматизации
- **Среда программирования:** CODESYS V3.5

### Технические характеристики

| Параметр | Значение |
|----------|----------|
| Процессор | ARM Cortex-A9, 800 МГц |
| ОЗУ | 512 МБ DDR3 |
| Flash | 8 ГБ eMMC |
| ОС | Linux (RT-патч) |
| Цикл выполнения | от 1 мс |
| Дискретные входы | 12 быстрых (FDI, до 95 кГц) |
| Дискретные выходы | 18 релейных (DO) |
| Аналоговые входы | зависит от модификации (0–4 AI) |
| Аналоговые выходы | зависит от модификации (0–2 AO) |

### Интерфейсы связи
- **Ethernet:** 2 × RJ-45 (10/100 Мбит, раздельные MAC-адреса)
- **RS-485:** 2 порта (Modbus RTU/ASCII Master/Slave)
- **USB:** 1 × USB Host (для подключения накопителей)
- **microSD:** слот для карты памяти

### Коммуникационные протоколы
- Modbus RTU/TCP (Master/Slave)
- OPC UA Server
- MQTT
- HTTP/FTP (веб-визуализация CODESYS)
- Поддержка ОВЕН Cloud

### Модули расширения
ПЛК210 работает с модулями серии **Мх210** по Ethernet:
- **Мх210-101** — 16 DI (дискретных входов)
- **Мх210-201** — 8 DO (релейных дискретных выходов)
- **Мх210-301** — 8 AI (аналоговых входов, 4–20 мА / 0–10 В)
- **Мх210-401** — 4 AO (аналоговых выходов)
- Другие модификации для термопар, термосопротивлений, счётчиков

### Модификации

| Артикул | DI | DO | AI | AO | Особенности |
|---------|----|----|----|----|-------------|
| ПЛК210-01-CS | 12 FDI | 18 DO (реле) | — | — | Базовая |
| ПЛК210-02-CS | 12 FDI | 6 DO (транз.) | 4 AI | 2 AO | С аналогом |
| ПЛК210-03-CS | 12 FDI | 12 DO (транз.) | — | — | Транзисторные выходы |

### Программирование
- Все языки IEC 61131-3 через CODESYS V3.5
- Target Package устанавливается в CODESYS IDE
- Веб-визуализация через встроенный Web-сервер
- Поддержка библиотек CODESYS: Modbus, Util, SysFile, CAA и др.
- Возможность написания пользовательских расширений на C (через CODESYS Application Composer)

### Типовые применения
- Управление вентиляцией (HVAC)
- Управление насосными станциями
- Котельная автоматика
- Системы водоподготовки
- Управление конвейерными линиями
- Системы диспетчеризации (в связке с SCADA)

## 2. REGUL R500

### Общие сведения
- **Производитель:** ООО «Прософт-Системы» (Россия, Екатеринбург)
- **Класс:** модульный промышленный контроллер для построения распределённых систем управления
- **Среда программирования:** Astra IDE
- **Позиционирование:** замена зарубежных DCS-контроллеров (Honeywell, Emerson, Yokogawa)

### Технические характеристики

| Параметр | Значение |
|----------|----------|
| Процессор (CU 00 071) | Intel Core, высокопроизводительный |
| Цикл выполнения | от 1 мс |
| Количество каналов I/O | до 10 000+ на контроллер |
| Крейт | масштабируемый с дискретностью в один модуль |
| Горячая замена | поддерживается для всех модулей |

### Архитектура

```
┌─────────────────────────────────────┐
│           Крейт (Backplane)         │
│  ┌────┐ ┌────┐ ┌────┐ ┌────┐       │
│  │ PS │ │ CU │ │ CM │ │I/O │ ...   │
│  │Пит.│ │ CPU│ │Связь│ │Мод.│       │
│  └────┘ └────┘ └────┘ └────┘       │
└─────────────────────────────────────┘
```

- **PS** — модуль питания (резервируемый)
- **CU** — модуль центрального процессора (резервируемый)
- **CM** — коммуникационные модули
- **I/O** — модули ввода-вывода

### Резервирование (REGUL R500S)
- Архитектура **1oo2D** (One out of Two with Diagnostics)
- Два процессорных модуля работают параллельно
- Бесударное переключение при отказе ведущего
- Резервирование шин связи
- Предназначен для непрерывных процессов (нефтегаз, энергетика, химия)

### Коммуникационные возможности
- **Ethernet:** множество портов (стандартный, резервируемый)
- **Протоколы:** Modbus RTU/TCP, OPC UA, МЭК 60870-5-101/104, МЭК 61850, Profibus DP, HART
- **Резервирование:** дублирование каналов связи

### Модули ввода-вывода
- DI (дискретные входы, 24В, с диагностикой)
- DO (дискретные выходы, релейные и транзисторные)
- AI (аналоговые входы: 4–20 мА, 0–10 В, термопары, RTD)
- AO (аналоговые выходы: 4–20 мА, 0–10 В)
- Специализированные модули (вибрация, высокоскоростной счёт)

### Особенности
- Конструкция устойчива к вибрации и ударам
- Расширенный температурный диапазон (−40..+70°C для отдельных модулей)
- Соответствие российским стандартам, входит в Реестр отечественного ПО
- Импортозамещение DCS-контроллеров

## 3. TREI-5B

### Общие сведения
- **Производитель:** ООО «ТРЭИ» (Россия, Чебоксары)
- **Класс:** модульный ПЛК для систем автоматизации и телемеханики
- **Среда программирования:** Unimod PRO
- **Серии:** TREI-5B-04, TREI-5B-05

### Архитектура

```
┌──────────────────────────────────────┐
│        Магистраль ST-BUS             │
│  ┌────┐ ┌────┐ ┌────┐ ┌────┐       │
│  │ БП │ │ CPU│ │Связь│ │I/O │ ...   │
│  └────┘ └────┘ └────┘ └────┘       │
└──────────────────────────────────────┘
```

### Шина ST-BUS
- Собственная внутренняя шина ТРЭИ
- Высокоскоростной обмен между модулями
- Поддержка горячей замены модулей

### Интерфейсы связи
- Ethernet (10/100 Мбит)
- RS-485, RS-232
- Оптические порты (для кольцевых сетей)

### Коммуникационные протоколы
- Modbus RTU/TCP
- МЭК 60870-5-101/104
- Собственные протоколы ТРЭИ

### Программирование
- Unimod PRO — CASE-система, поддерживающая языки IEC 61131-3
- Полностью отечественная разработка
- Исполнительная система: целевая задача (ядро) + задачи связи

### Типовые применения
- Телемеханика в электроэнергетике (подстанции, ЛЭП)
- АСУ ТП нефтегазовых объектов (КТП, ДНС, УКПН)
- Автоматизация объектов ЖКХ (тепловые пункты, насосные)

## 4. ЭЛСИ-ТМК

### Общие сведения
- **Производитель:** ООО «ЭлеСи» (Россия, Томск)
- **Класс:** модульный ПЛК для телемеханики и локальных систем автоматики
- **Позиционирование:** высоконадёжный контроллер с расширенными коммуникационными возможностями

### Технические характеристики

| Параметр | Значение |
|----------|----------|
| Точки ввода-вывода | до 10 000 на 1 контроллер |
| Горячая замена | поддерживается |
| Резервирование питания | да |
| Резервирование каналов связи | да |
| Дублирование корзин I/O | да |

### Модульный состав
- **Процессорные модули (TN)** — центральные модули с коммуникационными портами
- **Модули DI** — дискретные входы (с фильтрацией дребезга, поддержка низкокачественных сигналов)
- **Модули DO** — дискретные выходы
- **Модули AI** — аналоговые входы (4–20 мА, RTD, термопары)
- **Модули AO** — аналоговые выходы
- **Коммуникационные модули TN** — с различными интерфейсами (Ethernet, RS-485, оптика)

### Коммуникационные возможности
- МЭК 60870-5-101 (RS-485, последовательные каналы)
- МЭК 60870-5-104 (TCP/IP)
- Modbus RTU/TCP
- Собственные протоколы ЭлеСи
- Поддержка сигналов ввода с низким качеством (специальная обработка)
- Встроенная функция гарантированной доставки данных

### Резервирование
- Резервирование модулей питания
- Резервирование каналов связи
- Дублирование корзин ввода-вывода
- Возможность применения в TMR-архитектурах (мажоритарное 2oo3) [*уточнить — по данным pta-expo.ru*]

### Применение
- **Телемеханика** — основное назначение (нефтепроводы, газопроводы, электросети)
- Системы диспетчерского управления (SCADA)
- Локальные системы автоматики (компрессорные, насосные, УКПН)
- Объекты «Транснефть», «Газпром», электросетевые компании
