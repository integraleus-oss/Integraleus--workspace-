## 1. CODESYS

### Общие сведения
- **Производитель:** 3S-Smart Software Solutions GmbH (Германия)
- **Текущая версия:** CODESYS V3.5 (актуальна SP19+)
- **Лицензия:** IDE бесплатна, рантайм лицензируется производителем ПЛК
- **Платформа:** Windows

### Поддерживаемые языки
Все 5 языков IEC 61131-3: **ST, LD, FBD, IL, SFC** + расширение **CFC** (Continuous Function Chart)

### Возможности

**Редактор кода:**
- Подсветка синтаксиса, автодополнение
- Навигация по POU, перекрёстные ссылки
- Поддержка ООП (IEC 61131-3 3rd edition): классы, интерфейсы, наследование, полиморфизм

**Отладка:**
- Онлайн-мониторинг переменных в реальном времени
- Пошаговая отладка (Step Into, Step Over, Step Out) для ST
- Точки останова (Breakpoints)
- Просмотр стека вызовов
- Force-значения (принудительная установка)
- Отображение HEX/DEC
- Осциллограф (Trace) — запись и отображение переменных во времени

**Визуализация:**
- Встроенный редактор визуализации (HMI)
- Графические примитивы: кнопки, индикаторы, графики, тренды
- **TargetVisualization** — отображение на панели оператора
- **WebVisualization** — доступ через браузер (HTML5)
- **CODESYS Remote TargetVisu** — удалённый доступ

**Симуляция:**
- **CODESYS Control Win V3** — виртуальный контроллер на ПК
- Позволяет отлаживать программу без реального оборудования
- Поддерживает визуализацию, Modbus, OPC UA
- Полноценная эмуляция рантайма

**Коммуникации:**
- Modbus RTU/TCP (Master/Slave)
- OPC UA (Server/Client)
- EtherCAT Master
- CANopen
- PROFINET (опционально)
- MQTT (через библиотеку)

**Библиотеки:**
- Менеджер библиотек с онлайн-репозиторием
- Стандартные библиотеки: Standard, Util, IoStandard
- Функциональные блоки ПИД-регулирования, обработки сигналов
- Возможность создания собственных библиотек

### Совместимость
Используется более чем 400 производителями ПЛК: ОВЕН, WAGO, Beckhoff (TwinCAT основан на CODESYS V2), ABB, Schneider (SoMachine), Eaton, IFM и др.

## 2. Astra IDE (РегЛаб)

### Общие сведения
- **Производитель:** ООО «Прософт-Системы» / «РегЛаб» (Россия, Екатеринбург)
- **Назначение:** среда разработки для контроллеров REGUL R500, R200, R600
- **Платформа:** Windows

### Поддерживаемые языки
Все 5 языков IEC 61131-3: **ST, LD, FBD, IL, SFC** + **CFC**

### Возможности
- Полная поддержка стандарта IEC 61131-3
- Конфигурирование аппаратной части (модули I/O, коммуникации)
- Настройка резервирования (1oo2D для R500S)
- Конфигурирование промышленных протоколов: Modbus, МЭК 60870-5-104, МЭК 61850, OPC UA
- Отладка онлайн с мониторингом переменных
- Встроенный симулятор
- Управление проектами, контроль версий
- Интеграция со SCADA-системами

### Особенности
- Российская разработка, входит в Реестр отечественного ПО
- Тесная интеграция с аппаратной платформой REGUL
- Поддержка резервированных архитектур (горячий резерв, бесударное переключение)
- Инструменты диагностики и обслуживания контроллеров

## 3. Полигон (ОВЕН)

### Общие сведения
- **Производитель:** ОВЕН (Россия, Москва)
- **Назначение:** упрощённая среда для программирования контроллеров ОВЕН ПЛК110 и аналогичных
- **Платформа:** Windows
- **Статус:** устаревшая, для новых контроллеров (ПЛК210, ПЛК200) используется CODESYS V3.5

### Поддерживаемые языки
Ограниченный набор: **FBD** (основной), элементы **ST**

### Возможности
- Графический редактор FBD
- Библиотека функциональных блоков (таймеры, счётчики, ПИД, математика)
- Простая конфигурация I/O
- Загрузка программы в контроллер
- Онлайн-мониторинг
- Низкий порог входа — подходит для начинающих

### Замечание
Для современных контроллеров ОВЕН (ПЛК210, ПЛК200, СПК1xx) ОВЕН рекомендует **CODESYS V3.5** с пакетами поддержки (Target Package).

## 4. ISaGRAF

### Общие сведения
- **Производитель:** ICS Triplex (ранее Altersys), сейчас Rockwell Automation
- **Назначение:** среда разработки для промышленных контроллеров, особенно в энергетике и телемеханике
- **Платформа:** Windows

### Поддерживаемые языки
Все 5 языков IEC 61131-3: **ST, LD, FBD, IL, SFC**

### Возможности
- Кроссплатформенный рантайм (ISaGRAF Runtime работает на Linux, QNX, VxWorks, Windows CE)
- Компактный исполняемый код
- Поддержка промышленных протоколов: Modbus, МЭК 60870-5-101/104, DNP3
- Горячее резервирование
- Используется в контроллерах ФИОРД (Россия) для задач телемеханики в энергетике

### Особенности
- Ориентирован на задачи телемеханики и энергетики
- Относительно закрытая экосистема
- Поддержка протоколов МЭК 60870-5-101/104 «из коробки»

## 5. Beremiz

### Общие сведения
- **Тип:** открытая (open-source) среда разработки
- **Лицензия:** LGPL v2.1+
- **Язык разработки:** Python
- **Платформа:** Windows, Linux
- **Сайт:** https://beremiz.org, документация: https://sm1820.github.io/beremiz/

### Поддерживаемые языки
Все 5 языков IEC 61131-3: **ST, LD, FBD, IL, SFC**

### Возможности
- Полная реализация IEC 61131-3
- Генерация кода на C → компиляция → загрузка в целевое устройство
- Поддержка CANopen, Modbus
- Встроенный PLCOpen Editor (графический редактор для FBD, LD, SFC)
- Текстовый редактор с подсветкой для ST и IL
- Отладка в реальном времени
- Кроссплатформенность (работает на ПК, Raspberry Pi, промышленных Linux-системах)

### Применение в России
- Используется для контроллеров **BRIC** (SM1820)
- Адаптирован для платформы **YAPLC** (Yet Another PLC)
- Документация на русском: https://bric-beremiz.readthedocs.io/ru/latest/

### Ограничения
- Нет встроенной визуализации HMI (в отличие от CODESYS)
- Меньшая экосистема библиотек
- Требует навыков для настройки целевой платформы

## 6. MasterSCADA 4D

### Общие сведения
- **Производитель:** ИнСАТ (Россия, Москва)
- **Назначение:** единая среда для SCADA и программирования контроллеров
- **Платформа:** Windows

### Поддерживаемые языки
**ST, FBD** (основные), элементы графического программирования

### Возможности
- Единый проект: SCADA + контроллер (общее дерево объектов)
- Визуальное проектирование алгоритмов в FBD
- Встроенная SCADA с трендами, мнемосхемами, архивированием
- Поддержка OPC UA, Modbus, МЭК 60870-5-104
- Веб-интерфейс оператора
- Поддержка контроллеров: Fastwel CPM723/CPM903, ОВЕН (через Modbus)

### Особенности
- Российская разработка, Реестр отечественного ПО
- Концепция «Единой среды» — нет необходимости в отдельных IDE для SCADA и ПЛК
- Объектно-ориентированный подход к проектированию

## 7. TIA Portal (Siemens)

### Общие сведения
- **Производитель:** Siemens AG (Германия)
- **Назначение:** единая среда для программирования ПЛК Siemens (S7-1200, S7-1500, S7-300/400), HMI, приводов
- **Платформа:** Windows
- **Текущая версия:** TIA Portal V19

### Поддерживаемые языки
**LAD (LD), FBD, SCL (ST), STL (IL), GRAPH (SFC), CFC**

### Возможности
- Полная интеграция: ПЛК + HMI + приводы + Safety в одном проекте
- PLCSIM — симулятор контроллера
- PLCSIM Advanced — расширенный симулятор с виртуальным Ethernet
- Профессиональная отладка: watch tables, forcing, tracing
- Safety Integrated — программирование функций безопасности (F-CPU)
- Интеграция с Git (с V17)

### Замечание
TIA Portal — проприетарная среда, работает только с контроллерами Siemens. Лицензия платная.

## 8. Unimod PRO (ТРЭИ)

### Общие сведения
- **Производитель:** ООО «ТРЭИ» (Россия, Чебоксары)
- **Назначение:** CASE-система для программирования контроллеров TREI-5B
- **Платформа:** Windows (возможен запуск под другими ОС)

### Поддерживаемые языки
Языки IEC 61131-3: **ST, FBD, LD, SFC**

### Возможности
- Инструментальная система + исполнительная система (ядро для контроллера)
- Задачи связи для интеграции с верхним уровнем
- Поддержка шины ST-BUS (собственная шина ТРЭИ)
- Полностью отечественное ПО
- Конфигурирование модулей ввода-вывода
- Отладка и мониторинг
