## 1. Siemens WinCC V8.0

**Полное название:** SIMATIC WinCC V8.0
**Предыдущая версия:** WinCC V7.5
**Документы-источники:**
- WinCC V8.0 Architectures (Siemens © 2024, 26 стр.)
- ST 80 Catalog — SIMATIC WinCC V7/V8 (Siemens © 2024, 472K символов)

### 1.1 Что это

PC-based SCADA-система для визуализации и управления процессами, производственными потоками, машинами и установками во всех отраслях. Часть Totally Integrated Automation (TIA).

### 1.2 Поддерживаемые ОС

- Windows 11 Professional/Enterprise (64-bit)
- Windows 10 Professional/Enterprise (64-bit)
- Windows Server 2022 Standard/Datacenter (64-bit)
- Windows Server 2019 Standard/Datacenter (64-bit)

### 1.3 Лицензирование PowerTags

**PowerTags** = data points, подключённые к контроллерам/источникам через WinCC channel. Один PowerTag = до 32 алармов.

Размеры лицензий: **128, 512, 2 048, 8 192, 65 536, 102 400, 153 600, 262 144** PowerTags

Internal tags (без подключения) — безлимитные в каждой лицензии.

Base license включает **512 archive tags**. Дополнительные archive tags: **1 500, 5 000, 10 000, 30 000** (countable, складываются).

**Типы лицензий:**
- **RT** = Runtime only
- **RC** = Runtime + Configuration (Engineering)
- **RT Client** = клиент без engineering
- **RC Client** = клиент с remote engineering

### 1.6 Архивирование

- MS SQL Server (встроенный, real-time DB)
- High-performance archiving текущих данных

### 1.7 Web-доступ

- **WebNavigator** — Windows-клиент (WinCC Viewer RT), до 150 подключений
- **WebUX** — HTML5 браузер, до 100 подключений (не hard limit)
- **WebDiagnostics** — лицензия на клиенте

### 1.8 Redundancy

- Redundancy Package = 2 лицензии
- Два сервера работают параллельно
- При сбое: автоматическое перенятие архивирования messages, process data, user archives (Recipe)
- Автоматическая синхронизация после восстановления
- Redundant server pairs поддерживают UNI и MULTI Clients

### 1.9 IT/OT интеграция

- **REST Connector:** push data to IT-apps, alarm notifications, data exchange
- **REST API:** внешние приложения → WinCC (read config, read/write values)
- **Cloud Dashboarding:** multi-factory → HQ
- **WinCC Connectivity Pack** — необходим для REST/OPC-DA Provider

### 2.1 Позиционирование

Для приложений с:
- Высокой степенью клиентской адаптации
- Большими и/или сложными проектами
- Специфическими системными функциями
- **Vendor-agnostic** (подключение любых ПЛК, но оптимизирован для Siemens)

### 2.2 Архитектура

- Client-server с функциональным разделением на процессы (**Managers**)
- Event-oriented system
- Load distribution на несколько компьютеров
- **Hot Standby Redundancy**
- **Disaster Recovery System** (Hot Standby + Distributed Systems)
- **Multi-server distributed до 2 048 систем**
- Гетерогенные ОС одновременно
- Multi-monitor, Multi-login, Multi-user
- Docker containers, VMware Cluster (HA) ESXi

### 2.3 Масштаб

- **10 000 000+ тегов**

- Object-oriented data model с наследованием, группами, type-in-type

### 2.4 Поддерживаемые ОС

### 2.7 Визуализация

### 4.6 Архитектуры АСУ ТП

5. **РСУ (распределённая)** — множество МРВ, каждый дублирует функции, система работает до последнего ПК
6. **Телемеханика** — Micro TRACE MODE на удалённом объекте, технология гарантированной доставки при нестабильной связи

### 4.13 Регуляторы (из Руководства, детально)

**PID** — классический ПИД-регулятор с ограничением MIN/MAX и anti-windup (прекращение накопления интегральной составляющей при насыщении). Период t в секундах.

**PID2** — расширенный PID с 16-битным MODE:
- Биты 0-1: запрет увеличения интегральной составляющей
- Бит 5: зона нечувствительности (16 предустановленных коэффициентов)
- Бит 7: сглаживание выхода
- Бит 8: ручной режим (Q = IN2) с безударным переходом
- Бит 11: обнуление интегральной составляющей
- Бит 12: фиксация интегральной + clipping
- Бит 15: отключение дифференциальной составляющей

**PDD / PDD2** — ПДД-регулятор (пропорционально-дифференциальный со второй производной)

**FZCTR** — нечёткий регулятор (fuzzy logic):
- 3 категории рассогласования: большое (g), среднее (n), малое (s)
- Настраиваемые коэффициенты KG, KN, KS (по умолч. 0.3, 0.2, 0.05)
- Блок SFZ для индивидуальных настроек диапазонов

**PREG** — трёхпозиционный регулятор с гистерезисом

**OBJ** — модель объекта управления (апериодическое звено 1-го порядка + запаздывание + помехи). Для отладки регуляторов.

**QREG** — оценка качества регулирования: QdA (макс. отклонение), QT (время рассогласования), QS (интегральное отклонение)

**TRACT** — обработка сигнала: зона нечувствительности, экспоненциальное сглаживание, ограничение
