# Аналитика по артикулам из таблицы компонентов

**Переработанная версия на основе полного архива изменений**

## Цель документа

Для каждого артикула из исходной таблицы показать:
1. базовый функционал компонентов, входящих в состав решения, **до 2024 года**;
2. развитие функционала и новые возможности, появившиеся **в 2024–2026 гг.**

## Источники

- исходная таблица артикулов и компонентов;
- аудит `Alpha.Server / Alpha.Historian` за 2021–2026;
- архив release notes / email-рассылок по компонентам платформы;
- локальные конспекты модулей Alpha Platform.

## Важное замечание по структуре

В этой редакции оставлены **только те компоненты, по которым в материалах удалось зафиксировать содержательные изменения**.

Это значит:
- компоненты без явного changelog не перечисляются отдельно;
- документ стал короче, но точнее;
- акцент смещён на те элементы, которые действительно добавляли новую ценность артикулу в 2024–2026 гг.

## Нормализация терминов

В исходной таблице используются старые и внутренние имена. В документе они интерпретируются так:
- `Server`, `Server.Core` → **Alpha.Server**
- `Historian`, `History.H` → **Alpha.Historian**
- `History.SQL` → SQL-хранилища / модуль истории Alpha.Server
- `DevStudio`, `Developer.Studio` → **Alpha.DevStudio**
- `WebViewer`, `HMI-WebViewer` → **Alpha.HMI.WebViewer**
- `HMI.Alarms`, `Alarms` → **Alpha.HMI.Alarms**
- `HMI.Trends`, `Trends` → исторический трендовый контур Alpha.HMI; в актуальной линейке графический функционал частично смещён в **Alpha.HMI.Charts** / встроенные графики Alpha.HMI
- `RMap` → **Alpha.RMap**

# 1. Артикулы одиночно стоящего АРМ

Артикулы:
- `TN-CPA-ARM-5K`
- `TN-CPA-ARM-10K`
- `TN-CPA-ARM-25K`

## 1.1. Общий вывод по семейству ARM

Три ARM-артикула имеют близкий функциональный состав. Их главное отличие — лицензируемая ёмкость ядра:
- `TN-CPA-ARM-5K` → `Server.Core 5000`
- `TN-CPA-ARM-10K` → `Server.Core 10000`
- `TN-CPA-ARM-25K` → `Server.Core 25000`

Следовательно:
- **функциональный контур** у решений практически общий;
- **масштаб внедрения** определяется серверной лицензией и числом обслуживаемых тегов.

## 1.4. Итог по ARM-артикулам

### TN-CPA-ARM-5K
- Полноценный одиночно стоящий АРМ с визуализацией, тревогами, графиками, веб-доступом, диагностикой, серверной логикой и широким набором протоколов обмена.
- Главный прирост ценности в 2024–2026 обеспечен развитием `Alpha.Server`, `Alpha.HMI`, `Alpha.HMI.WebViewer`, `Alpha.HMI.Alarms`, `Alpha.HMI.Trends / Charts`, а также коммуникационных модулей и истории.
- Ограничение определяется лицензией `Server.Core 5000`.

### TN-CPA-ARM-10K
- Функционально соответствует `TN-CPA-ARM-5K`.
- Отличается лицензией `Server.Core 10000`, то есть рассчитан на более крупный масштаб проекта.

### TN-CPA-ARM-25K
- Функционально соответствует двум предыдущим артикулам.
- Отличается лицензией `Server.Core 25000` и является самым ёмким вариантом семейства.

# 2. Артикулы сервера истории

Артикулы:
- `TN-CPA-PHD500`
- `TN-CPA-PHD1K`

## 2.1. Общий вывод по семейству PHD

Оба артикула имеют один и тот же функциональный состав:
- `Historian`
- `Tools`
- `RMap`
- `Licensing`
- `DevStudio`

Главное различие — лицензируемая ёмкость архива:
- `TN-CPA-PHD500` → `Historian 500`

## 2.3. Итог по PHD-артикулам

### TN-CPA-PHD500
- Сервер истории на базе `Alpha.Historian` с SQL-доступом через `Alpha.RMap` и инженерным сопровождением через `Alpha.DevStudio`.
- Главный прирост ценности в 2024–2026 — переход на поколение `Alpha.Historian 4.x`.
- Отличие от PHD1K — только в лицензируемой ёмкости архива.

### TN-CPA-PHD1K
- Функционально соответствует `TN-CPA-PHD500`.
- Отличается лицензией `Historian 1000`.
- Все новые возможности `Alpha.Historian 4.x` применимы в полном объёме.

# 3. Ключевые выводы

1. **ARM-артикулы развивались главным образом за счёт серверного и HMI-контура**, а не за счёт изменения состава лицензий.
2. **Наиболее заметный прирост в 2024–2026 гг. дали**:
   - `Alpha.Server`;
   - `Alpha.HMI` и `Alpha.HMI.WebViewer`;
   - `Alpha.HMI.Alarms`;
   - `Alpha.HMI.Trends / Alpha.HMI.Charts`;
   - `Alpha.DevStudio`;
   - коммуникационные модули `IEC-104`, `Modbus`, `OPC UA`, `SNMP`, `MQTT`, `SQL Connector`, `HUB`, `Snapshot`.
3. **PHD-артикулы получили основной прирост ценности за счёт перехода к Alpha.Historian 4.x** и ускорения SQL-доступа через `Alpha.RMap`.
4. **Различия внутри каждого семейства артикулов — в первую очередь масштабные, а не функциональные**:
   - ARM: 5K / 10K / 25K;
   - PHD: 500 / 1000.
