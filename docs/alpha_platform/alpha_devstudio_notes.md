# Alpha.DevStudio 4.1 — Конспект

**Продукт:** Среда разработки для проектирования систем автоматизации (АСУ ТП)
**Разработчик:** АО «Атомик Софт» (Automiq)
**Версия:** 4.1
**ОС:** Windows 10/11, Server 2012-2022, Ubuntu, Astra Linux, Альт, РЕД ОС

## Назначение
- Описание физической и логической структуры объектов автоматизации
- Создание схем развёртывания
- Дистрибуция конфигураций
- Совместная разработка (SVN, Git)

## Установка
- Требуется .NET 8.0 Runtime
- Linux: + libgdiplus, SVN
- CLI: `devstudio.cli.exe` для автоматизации без GUI

## Ключевые концепции

### Решения и проекты
- **Решение** — контейнер для проектов и модулей
- **Проект** — редактируемая единица
- **Модуль** (*.binom) — опубликованный проект, неизменяемый
- Системные модули: Om.System, Om.Server, Om.OPC, Om.PLC, Om.IEC60870, Om.BACnet, Om.EtherNetIP, Om.HMI

### Домен и компоненты
- **Alpha.Domain** — инфраструктурная подсистема, объединяет компоненты
- **Alpha.Server** — серверный компонент, исполняет приложения
- **Alpha.AccessPoint** — АРМ оператора
- **Контроллеры (ПЛК)** — Modicon, OMRON, REGUL, SIMATIC, ТРЭИ, универсальный
- **Исполняющие компоненты** — сторонние системы

### Объекты и аспекты
- **Объект** — набор данных и правил (датчики, трубопроводы и т.д.)
- **Аспект** — угол зрения (представление объекта в разных контекстах)
- **Сигналы**: параметры (постоянные) и события (моментальные)
- **Сокеты** — группы сигналов для связывания
- **Связи** — передача данных между сигналами (VQT, побитовая, качество, статус)

### Вычисления
- **Процедуры** (обработчики событий) — код на Alpha.Om
- **Формулы** — автопересчёт при изменении входов
- **Таймеры** — фиксированные и настраиваемые
- **Расписания** — cron-выражения (6 полей)
- **DLL** — внешние функции

### Типы данных
void, bool, int1-int8, uint1-uint8, float, double, time, string + массивы

## Передача данных
- Карты адресов для каждого протокола
- Серверные и клиентские логические адаптеры
- Объединение в сети (Ethernet, последовательная шина, файловый обмен)
- Категории данных (разные периоды опроса)
- Множества — группы сигналов для фильтрации (мёртвая зона, мин. интервал)

## Резервирование
- **Дублирование** — оба работают одновременно
- **Горячее резервирование** — резервный в ожидании
- Применимо к: Alpha.Server, АРМ, серверы истории, исполняющие компоненты

## Буферизация
- По Alpha.Link при разрыве связи
- Транзакционная модель Alpha.Server 6.0+
- Data Buffer в ПЛК

## События и агрегация
- Условия генерации: перечисление, по уровню, отклонение, логическое, сообщение
- Подусловия: значение, сообщение, важность, звук, квитирование, задержка
- Агрегатор — обобщённая информация по ветке (активность, количество, важность)

## Серверы истории
- Alpha.Historian (собственный)
- MS SQL Server
- PostgreSQL

## CLI (devstudio.cli.exe)
- `compile` — компиляция решения/проекта
- `build` — построение конфигураций (status/build/rebuild/clean)
- `deploy` — развёртывание
- `deploy-status` — статус конфигураций
- `deploy-commit` — подтверждение стабильности
- `deploy-rollback` — откат
- `deploy-select` — применение конкретных версий
- `publish` — публикация модулей
