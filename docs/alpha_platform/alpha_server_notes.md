# Конспект модулей Alpha.Server

## Data Buffer

**Название и назначение:** Модуль Data Buffer - буфер данных для хранения значений сигналов в памяти с возможностью доступа к ним через специальные параметры адреса.

**Ключевые возможности:**
- Хранение текущих и предыдущих значений сигналов
- Доступ к значениям через параметр MemberId
- Работа с различными типами данных
- Буферизация данных в памяти

**Поддерживаемые режимы:** Работает как служебный модуль в составе Alpha.Server

**Ограничения / важные нюансы:**
- Данные хранятся только в памяти
- При перезапуске сервера данные теряются
- Требует настройки свойства 5000 (Address) для сигналов

**Версия:** Входит в состав Alpha.Server 6.4

---

## DTS Client

**Название и назначение:** Модуль DTS Client предназначен для обмена данными между Alpha.Server и подсистемой передачи данных СВУ «Портал» (ППД).

**Ключевые возможности:**
- Отправка значений сигналов Alpha.Server в ППД
- Приём значений от ППД и сохранение в сигналы Alpha.Server
- Поддержка нескольких каналов передачи данных
- Работа с групповыми значениями

**Поддерживаемые режимы:** Client (клиент для ППД)

**Ограничения / важные нюансы:**
- Работает только в ОС семейства Linux
- Одновременно может функционировать только один экземпляр модуля
- Настройка через удаленные сервисные приложения с ОС Windows
- Использует UDP порт по умолчанию 49000

**Версия:** 6.4.28

## EtherNet/IP Scanner

**Название и назначение:** Модуль EtherNet/IP Scanner - коммуникационный модуль для обмена данными между Alpha.Server и ПЛК по протоколу EtherNet/IP.

**Ключевые возможности:**
- Сбор данных с ПЛК ControlLogix 5575 (Rockwell Automation)
- Подача команд в ПЛК
- Синхронизация времени ПЛК
- Подача импульсных команд
- Обмен данными при резервировании ПЛК и каналов связи
- Оптимизация трафика через группы опроса

**Поддерживаемые режимы:** Scanner (сканер/клиент)

**Ограничения / важные нюансы:**
- Совместим только с ПЛК ControlLogix 5575
- Использует TCP протокол в режиме запрос-ответ
- Требует лицензирования на количество сигналов
- Поддержка битовых операций AND/OR

**Версия:** 6.4.28

## FINS Client

**Название и назначение:** Модуль FINS Client - коммуникационный модуль для обмена данными между Alpha.Server и программируемым контроллером OMRON (ПЛК) по протоколу FINS.

**Ключевые возможности:**
- Сбор данных с ПЛК OMRON
- Подача команд в ПЛК
- Работа с различными областями памяти ПЛК (CIO, WR, HR, AR, DM, TIM, CNT, EM)
- Поддержка нескольких ПЛК одновременно
- Категории данных для оптимизации опроса

**Поддерживаемые режимы:** Client (клиент)

**Ограничения / важные нюансы:**
- Работает по UDP протоколу (порт 9600 по умолчанию)
- Использует команды MEMORY AREA WRITE и MULTIPLE MEMORY AREA READ
- В режиме РЕЗЕРВ выполняет те же функции, что и в режиме РАБОТА
- Требует настройки адресов сети и узлов для источника и получателя

**Версия:** 6.4.28

## Write VQT

**Название и назначение:** Модуль Write VQT позволяет изменять свойства сигнала (Value, Quality, Timestamp) независимо друг от друга.

**Ключевые возможности:**
- Независимое изменение значения (Value)
- Независимое изменение качества (Quality)  
- Независимое изменение метки времени (Timestamp)
- Поддержка различных масок изменения
- Создание дочерних служебных сигналов для управления

**Поддерживаемые режимы:** Служебный модуль

**Ограничения / важные нюансы:**
- Требует добавления свойства 10000 со значением True для сигналов
- Создает дочерние сигналы (vqt_value, vqt_quality, vqt_timestamp, vqt_changemask)
- Поддержка двух форматов метки времени
- Маски от 32 и выше устанавливают текущее системное время

**Версия:** 6.4.28

## Модуль истории

**Название и назначение:** Модуль истории предназначен для сохранения данных о ходе технологического процесса в различные базы данных.

**Ключевые возможности:**
- Сохранение значений сигналов в БД (Alpha.Historian, MS SQL Server, PostgreSQL)
- Сохранение событий в БД (Alpha.Historian, PostgreSQL)
- Запрос сохраненных данных из БД
- Фильтрация значений при сохранении (мертвая зона, интервалы времени)
- Резервирование истории через несколько БД
- Периодическое повторное сохранение записей

**Поддерживаемые режимы:** Служебный модуль (запись в РАБОТА, чтение в РЕЗЕРВ)

**Ограничения / важные нюансы:**
- В режиме РЕЗЕРВ не сохраняет данные
- MS SQL Server только для истории значений (не событий)
- Требует настройки свойства 9001 (Historizing) для сигналов
- В Alpha.AccessPoint только чтение данных
- Поддержка PostgreSQL версий 9.5-17

**Версия:** 6.4.28
