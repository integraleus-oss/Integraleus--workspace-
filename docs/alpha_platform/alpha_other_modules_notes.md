# Конспекты модулей Альфа платформы

## 1. Alpha.Security — подсистема безопасности

**Название и версия:** Alpha.Security (версия не указана)

**Назначение:** Комплексная подсистема безопасности для обеспечения контроля доступа, аутентификации и аудита в системах автоматизации.

**Ключевые возможности:**
- Интеграция с LDAP-серверами для централизованной аутентификации
- Контроль целостности данных и системы
- Ведение аудиторских журналов событий безопасности
- Управление учетными записями пользователей
- Контроль доступа к ресурсам системы
- Поддержка различных методов аутентификации

**Архитектура / компоненты:**
- **LDAP-сервер** — служба каталогов для хранения учетных записей
- **Агент безопасности** — компонент для контроля доступа
- **SecurityConfigurator** — инструмент настройки безопасности
- **Модуль аудита** — система регистрации событий безопасности

**Ограничения / важные нюансы:**
- Требует правильной настройки LDAP для корректной работы
- Необходима настройка политик безопасности до ввода в эксплуатацию
- Производительность может снижаться при интенсивном аудите

**Поддерживаемые ОС:** Windows, Linux

## 2. Alpha.Reports — система отчётов

**Название и версия:** Alpha.Reports 1.1

**Назначение:** Комплексная система формирования, настройки и просмотра отчетов для анализа данных технологических процессов.

**Ключевые возможности:**
- Визуальный конструктор отчетов с drag-and-drop интерфейсом
- Конфигуратор сервера отчетов
- Библиотека готовых шаблонов отчетов
- Просмотрщик отчетов с возможностями печати и экспорта
- Планировщик автоматического формирования отчетов
- Поддержка различных источников данных

**Архитектура / компоненты:**
- **Конструктор отчетов** — инструмент создания шаблонов
- **Конфигуратор сервера** — настройка параметров сервера отчетов
- **Библиотека отчетов** — хранилище шаблонов и готовых отчетов
- **Просмотрщик отчетов** — клиентское приложение для просмотра
- **Сервер отчетов** — серверная часть системы

**Ограничения / важные нюансы:**
- Требуется настройка подключения к СУБД для хранения отчетов
- Производительность зависит от сложности отчетов и объема данных
- Некоторые функции могут быть недоступны в зависимости от лицензии

**Поддерживаемые ОС:** Не указано в документации

## 4. Alpha.AccessPoint — точка доступа

**Название и версия:** Alpha.AccessPoint (версия не указана)

**Назначение:** Централизованная точка доступа к данным различных источников в распределенных системах автоматизации.

**Ключевые возможности:**
- Агрегация данных от множественных источников Alpha.Server
- Единый интерфейс доступа к распределенным данным
- Кэширование данных для повышения производительности
- Балансировка нагрузки между источниками
- Отказоустойчивость при недоступности отдельных источников
- Поддержка различных протоколов обмена данными

**Архитектура / компоненты:**
- **Диспетчер подключений** — управление соединениями с источниками
- **Кэш данных** — временное хранилище для ускорения доступа
- **Маршрутизатор запросов** — распределение запросов по источникам
- **Модуль агрегации** — объединение данных от разных источников

**Ограничения / важные нюансы:**
- Не предназначена для сохранения данных, только для агрегации
- Производительность зависит от качества сетевых соединений
- Требует настройки правил маршрутизации для корректной работы

**Поддерживаемые ОС:** Windows, Linux

## 5. Alpha.Imitator — имитатор

**Название и версия:** Alpha.Imitator (версия не указана)

**Назначение:** Средство имитации технологических процессов и устройств для тестирования и отладки систем автоматизации.

**Ключевые возможности:**
- Имитация работы технологического оборудования
- Генерация тестовых данных различных типов
- Моделирование аварийных и нештатных ситуаций
- Поддержка различных протоколов связи
- Настраиваемые сценарии поведения устройств
- Интеграция с системами разработки и тестирования

**Архитектура / компоненты:**
- **Движок имитации** — ядро системы моделирования
- **Генератор данных** — формирование тестовых значений
- **Менеджер сценариев** — управление логикой поведения
- **Интерфейсы протоколов** — поддержка различных способов связи

**Ограничения / важные нюансы:**
- Предназначен исключительно для тестирования, не для продуктивного использования
- Точность имитации зависит от качества настроенных моделей
- Может потреблять значительные ресурсы при сложном моделировании

**Поддерживаемые ОС:** Windows, Linux

## 6. Alpha.RMap — удалённый мониторинг

**Название и версия:** Alpha.RMap (версия не указана)

**Назначение:** Компонент для предоставления оперативных значений, истории и событий в реляционном виде через SQL-интерфейс.

**Ключевые возможности:**
- Предоставление данных через SQL-запросы
- Представление оперативных значений в реляционном формате
- Доступ к историческим данным через стандартные SQL-операции
- Интеграция с внешними системами аналитики и отчетности
- Поддержка стандартных СУБД-интерфейсов
- Оптимизация запросов для повышения производительности

**Архитектура / компоненты:**
- **SQL-движок** — обработка запросов к данным
- **Адаптер данных** — преобразование внутренних структур в реляционные
- **Кэш запросов** — ускорение повторных обращений
- **Менеджер соединений** — управление подключениями клиентов

**Ограничения / важные нюансы:**
- Производительность зависит от сложности SQL-запросов
- Требует настройки индексов для оптимальной работы
- Ограничения на одновременное количество подключений

**Поддерживаемые ОС:** Не указано в документации

## 6. Alpha.Om — язык программирования платформы

**Название и версия:** Alpha.Om 1.4

**Назначение:** Язык для описания исполняемых элементов скриптов (процедур и формул), которые запускаются в компонентах Альфа платформы — Alpha.Server, Alpha.HMI, Alpha.DevStudio.

**Ключевые возможности:**
- Формулы (автоматический пересчёт при изменении аргументов) и процедуры (выполнение по событию)
- Типы данных: int1/2/4/8, uint1/2/4/8, float, double, bool, string, variant
- Арифметические, логические, битовые операции, операции сравнения
- Управляющие конструкции: if/else, while (циклы)
- Переменные и константы с областями видимости
- Относительная адресация: me, @1, @2 (навигация по иерархии объектов)
- Явное чтение аргументов (read) — управление триггерами пересчёта формул
- Явное и неявное преобразование типов (TypeConvert)
- Регистрозависимый язык
- Поддержка Unicode в строках (UTF16)
- Поддержка NaN, Infinity, отрицательный ноль

**Ограничения / важные нюансы:**
- ❌ Нет массивов (в версии 1.4; в презентации 2024-2025 заявлена поддержка массивов в вычислениях)
- ❌ Нет пользовательских структур (struct)
- ❌ Нет пользовательских функций
- ❌ Нет рекурсии
- Нет неявного преобразования double → float
- Язык НЕ является языком общего назначения — предназначен для промышленной логики

## 7. Alpha.Tools — инструменты диагностики

**Название и версия:** Alpha.Tools 1.6

**Назначение:** Набор инструментов для отладки и диагностики компонентов Альфа платформы.

**Ключевые возможности:**
- **OpcExplorer** — работа с OPC серверами по протоколам DA, AE, HDA, UA:
  - Подключение к серверам, просмотр тегов, чтение/запись значений
  - Пользовательский интерфейс в стиле Alpha.HMI
- **EventLogViewer** — просмотр журналов событий:
  - Фильтрация и поиск событий
  - Работа с журналами Alpha.Server

**Ограничения / важные нюансы:**
- Требуется документация Alpha.Tools версии 1.6 для корректной работы

---

## 8. Alpha.Trends — приложение просмотра трендов

**Название и версия:** Alpha.Trends 3.33

**Назначение:** Приложение для просмотра и анализа графиков оперативных и исторических данных в пунктах автоматизации и мониторинга.

**Ключевые возможности:**
- Просмотр графиков оперативных данных в реальном времени
- Просмотр графиков исторических данных
- Работа как самостоятельное приложение или как встраиваемый компонент
- Встраивание в Alpha.HMI проекты
- Подключение к локальным и удалённым серверам

**Архитектура:**
- Может работать standalone или embedded
- Установка: `C:\Program Files (x86)\Automiq\Alpha.Trends`
- Диагностические сообщения записываются в журнал приложений Windows

**Поддерживаемые ОС:** Windows (указана только Windows-установка)
