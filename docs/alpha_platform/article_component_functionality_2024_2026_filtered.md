# Аналитика по артикулам из таблицы компонентов

**Задача:** по каждому артикулу из исходной таблицы показать:
1. функционал компонентов, входящих в артикул, **до 2024 года**;
2. изменения и новые возможности **в период 2024–2026**.

**Исходные материалы:**
- Excel с артикулами и составом компонентов;
- аудит `Alpha.Server / Alpha.Historian` по релизам 2021–2026;
- локальные конспекты модулей Alpha.Platform.

## Важная оговорка по терминологии

В Excel используются **старые / внутренние названия** компонентов. Ниже они нормализованы к актуальной терминологии Alpha Platform там, где это возможно.

Примеры нормализации:
- `HMI-WebViewer` → **Alpha.HMI.WebViewer**
- `DevStudio` / `Developer.Studio` → **Alpha.DevStudio**
- `Historian` → **Alpha.Historian**
- `Security` → **Alpha.Security**
- `Server` / `Server.Core` → **Alpha.Server**
- `HMI.Alarms` / `Alarms` → **Alpha.HMI.Alarms** / функционал тревог Alpha.Platform
- `HMI.Trends` / `Trends` → исторически отдельный функционал трендов; в актуальной линейке корректнее трактовать как **графики в Alpha.HMI (alpha.hmi.charts)**

## Как читать документ

- Если по компоненту есть явные релизные изменения 2024–2026 в аудите, они перечислены в колонке «Что появилось / изменилось в 2024–2026».
- Если отдельные изменения в изученных материалах **не выделены**, это прямо указано.
- Для артикулов, имеющих одинаковый состав, повторяющиеся описания не переписываются заново: фиксируется, что состав функционально совпадает, а меняется только ёмкость лицензии.

# 1. Артикулы одиночно стоящего АРМ

Артикулы:
- `TN-CPA-ARM-5K`
- `TN-CPA-ARM-10K`
- `TN-CPA-ARM-25K`

## 1.1. Общий вывод по семейству ARM

Эти три артикула имеют **одинаковый функциональный состав**, но отличаются масштабом лицензии ядра:
- `TN-CPA-ARM-5K` → `Server.Core 5000`
- `TN-CPA-ARM-10K` → `Server.Core 10000`
- `TN-CPA-ARM-25K` → `Server.Core 25000`

То есть:
- **функционально** решения очень близки;
- **масштабно** отличаются допустимым объёмом тегов/сигналов и серверной лицензией.

# 3. Концентрированная выжимка: что реально нового появилось после 2024 года

## Для ARM-артикулов
Наиболее важные новые возможности 2024–2026:
- развитие **Alpha.Server** как ядра: буферизация, резервирование, фильтрация значений, диагностика, аутентификация, управление экземплярами;
- резкое усиление **OPC AE / тревожной подсистемы**: OAT, агрегаторы, задержка генерации, корректное отображение зон/подавления/квитирования;
- значимое развитие **OPC UA Client / Server**: масштабирование, сертификаты, политики безопасности, устойчивость после разрывов;
- развитие промышленных протоколов: **IEC-104**, **Modbus TCP/RTU**, **MQTT**, **SNMP**, **BACnet**, затем **MDLC Client**;
- развитие связки **TCP Server ↔ HUB** и обмена с **Alpha.AccessPoint**;
- улучшение поддержки графиков и исторических данных за счёт развития Historian / PostgreSQL / серверной части.

## Для PHD-артикулов
Наиболее важные новые возможности 2024–2026:
- выпуск **Alpha.Historian 4.0** как нового поколения сервера истории;
- формат БД **4x**;
- запись с любой меткой времени;
- повышение производительности, снижение накладных расходов по памяти и диску;
- новый мониторинг, CLI и новая модель конфигурации;
- централизованные политики через Alpha.Security;
- в 2026 — контроль допустимого отклонения временных меток для БД 4x.
