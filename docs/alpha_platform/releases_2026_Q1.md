## Ключевые изменения (топ-9 для руководства)

### 1. 🔴 Критическое повышение стабильности ядра (Alpha.Server 6.4.27–6.4.30)

Устранён целый класс проблем, приводивших к **аварийному завершению службы Alpha.Server на Linux**:
- Краш при обмене данными между серверами по протоколу Alpha.Link
- Краш при подключении модуля Security Client к агенту безопасности
- Краш при работе модуля логики с вычислениями на основании начальных значений
- Повышенная загрузка ЦП на Linux (исправлено в нескольких модулях: основной сервер, Modbus TCP Master)

**Влияние:** На промышленных объектах с Linux (Astra Linux) эти исправления напрямую влияют на непрерывность работы SCADA-систем.

### 2. 🔴 Капитальная переработка подсистемы резервирования

- Резервный переход выполняется **только когда оба сервера в режиме "РАБОТА"** — решена потеря данных в историю при переключении
- Устранены крахи резервного сервера при частых переключениях с Modbus RTU Master
- Исправлена репликация сигналов на резервный сервер после старта пары

**Влияние:** Потеря исторических данных при переходе на резерв — ранее скрытая проблема. Теперь механизм работает корректно.

### 3. 🟠 Новые режимы работы устройств в резерве (Универсальный модуль, 6.4.28)

Три режима поведения при переходе в РЕЗЕРВ:
- **Закрывать соединение** — полный разрыв до возврата в РАБОТУ
- **Поддерживать соединение** — связь есть, но данные не передаются
- **Получать данные** — модуль читает, но не пишет в устройства

**Влияние:** На критичных участках резервный сервер может уже иметь актуальные данные на момент переключения. Сокращает время перехода на резерв.

### 4. 🟠 Масштабирование OPC UA и ускорение отказоустойчивости

- Лимит OPC UA серверов увеличен **100 → 200** на один модуль
- Устранён дефект: модуль мог не пытаться восстановить связь до **1 ч 15 мин** после разрыва
- Переключение на резервный ПЛК ускорено с **5–10 секунд** до штатного времени
- Исправлена потеря буферизованных данных через межсетевой шлюз

**Влияние:** На крупных объектах с десятками ПЛК лимит в 100 серверов был узким местом.

### 5. 🟠 Alpha.DevStudio 4.2.0 — крупный релиз среды разработки

- **Описание рабочих мест (АРМ)** — реальные компьютеры + развёртывание конфигурации прямо на них
- Оптимизация: диаграммы >15 000 символов, таблицы >10 000 элементов
- Поддержка IEC 61850 — имя устройства в свойствах элемента
- Функция «Найти все ссылки» — быстрый поиск зависимостей

**Влияние:** Сокращение времени проектирования и развёртывания. Шаг к полной автоматизации деплоя.

### 6. 🟠 Alpha.Domain 1.5.3 — поддержка ARMv8 на Linux

Поддержка процессорной архитектуры ARMv8. Защита от аварийного завершения при несовпадении InstanceID.

**Влияние:** Расширение аппаратной совместимости в контексте импортозамещения (Байкал и др.).

### 7. 🟡 Усиление кибербезопасности (Alpha.Security 1.7.27–1.7.28)

- Новые политики паролей: проверка наличия логина/имени в пароле, настройка допустимых символов
- Сохранение сессии при потере связи с LDAP
- Устранены крахи агента безопасности
- Снято ограничение на импорт учётных записей

**Влияние:** Соответствие ФСТЭК/ФСБ. Стабильность авторизации операторов.

### 8. 🟡 Исправление детектирования потери TCP на Linux

Общее для всех TCP-модулей: ранее потеря связи на Linux не детектировалась своевременно.

**Влияние:** Оператор мог видеть «замёрзшую» картинку, не зная, что реальных данных нет. Критично для безопасности.

### 9. 🟡 Alpha.Historian 4.0.4 — фильтрация по отклонению меток времени

Возможность отбрасывать записи с некорректными метками времени.

**Влияние:** Защита целостности исторических данных от сбоев NTP/ПЛК.

6. **UX модернизация** — SVG иконки во всех HMI-компонентах, оптимизация крупных проектов
