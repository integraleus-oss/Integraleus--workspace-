# Alpha.Domain — Конспект

**Продукт:** Инфраструктурная подсистема Альфа платформы
**ОС:** Windows, Linux

## Назначение
Объединение компонентов (Alpha.Server, Alpha.AccessPoint, Alpha.Security) в единую среду исполнения — **домен**.

## Ключевые компоненты
- **Domain-агент** (Alpha.Domain.Agent) — объединение компонентов в домен
- **Net-агент** (Alpha.Net.Agent) — объединение компьютеров в сеть Alpha.Net

## Роли компьютеров
- **Серверные** — Alpha.Server, Alpha.Historian
- **Клиентские** — Alpha.AccessPoint, HMI-приложения
- **Инженерные** — только Alpha.DevStudio

## Установка
- Windows: MSI-файл, службы запускаются автоматически
- Linux: rpm/deb, службы нужно запустить вручную через systemctl

## Запуск от непривилегированного пользователя (Linux, v1.4.8+)
1. Создать пользователя + linger
2. Остановить все сервисы
3. Сменить порт Net-агента >10000
4. Обновить юнит-файлы (User)
5. Настроить server_launcher.sh
6. chown папок установки
7. Папка кэша в конфигурации

## Принципы
- Простые задачи (кластерное АРМ) — достаточно Net-агента
- Развёртывание из DevStudio — нужен домен + Alpha.Net
