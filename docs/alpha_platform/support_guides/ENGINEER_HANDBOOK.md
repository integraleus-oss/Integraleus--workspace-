# Справочник инженера Alpha Platform

Краткий справочник по настройке ключевых компонентов Alpha Platform с практическими инструкциями.

## Как настроить Alpha.Security

**Что нужно:**
- Microsoft Visual C++ 2015/2017/2019 Redistributable (x86/x64)
- Microsoft .NET Framework 3.5 и 4.6
- OPC .NET API 2.00 Redistributables 105.0
- OPC Core Components Redistributable (x64) 105.0
- OpenLDAP

**Шаги:**
1. Установить OpenLDAP согласно разрядности системы
2. Запустить ConfigurationMaster.exe от имени администратора
3. Настроить DCOM
4. Заполнить конфигурацию: адрес LDAP, параметры подключения
5. Настроить OPC DA сервер и сигналы для аудита
6. Сконфигурировать Alpha.Server для генерации событий

**Особенности:**
- Обязательно запускать ConfigurationMaster от имени администратора
- Настройка DCOM критична для корректной работы
- При настройке аудита создать динамические события для Alpha.Alarms

**Источник:** Alpha.Security стартовая инструкция.docx, Инструкция по настройке аудита.docx

## Как настроить Alpha.HMI.WebViewer с Docker

**Что нужно:**
- Ubuntu/Linux система
- Docker
- Apache2
- Установочный .deb пакет alpha.hmi.webviewer
- Сконфигурированный alpha.hmi.webviewer.xml
- Проект HMI

**Шаги:**
1. Установить Docker: `sudo apt install docker-ce`
2. Добавить пользователя в группу docker: `sudo usermod -a -G docker username`
3. Создать рабочую директорию: `mkdir /home/user/webviewer-container`
4. Поместить в директорию: .deb пакет, конфиг XML, проект HMI
5. Создать Dockerfile с настройкой Ubuntu + Apache + компонентов Alpha
6. Создать start.sh скрипт: `apachectl -k start & alpha.hmi.webviewer`
7. Собрать образ: `docker build -t custom_image .`
8. Запустить контейнер: `docker run -it -p 80:80 -p 8080:8080 custom_image`

**Особенности:**
- Необходимо установить множество зависимостей в контейнере
- Проброс портов 80 и 8080 для веб-доступа
- Требуется настройка языкового пакета ru

**Источник:** Запуск WebViewer в Docker.docx

## Как настроить Alpha.HMI.WebViewer с Apache/NGINX

**Что нужно:**
- Linux система (Ubuntu/Debian)
- Apache2 или NGINX
- Компоненты Alpha Platform: HMI, Charts, Tables, Security
- SSL сертификаты (при необходимости HTTPS)

**Шаги:**
1. Установить веб-сервер: `sudo apt install apache2` или `sudo apt install nginx`
2. Установить компоненты Alpha Platform: HMI, Charts, Tables, Security и их WebViewer варианты
3. Создать директорию: `/var/www/webviewer.local/html`
4. Настроить права пользователя на директорию
5. Создать и настроить виртуальный хост
6. Для HTTPS: создать самоподписанный сертификат или использовать существующий

**Особенности:**
- При создании самоподписанного сертификата использовать параметры: страна RU, регион Томск, организация Automiq, срок 365 дней
- Настройка прав доступа критична для корректной работы

**Источник:** Инструкция по установке NGINX и webviewer.docx, Настройка Apache и создание самоподписанных сертификатов на Astra.docx

## Как настроить Alpha.Domain

**Что нужно:**
- Установленный Alpha.Domain
- Права на редактирование конфигурационных файлов

**Шаги:**
1. Открыть файл alpha.net.agent.xml в C:\Program Files\Automiq\Alpha.Domain
2. В теге Alpha.Net.Agent прописать параметры:
   - Name (имя узла)
   - NetEnterPort (порт входа)
   - ParentAgentPort (при необходимости)
3. Настроить ChildAgent с указанием имени, адреса и порта дочерних узлов
4. Отредактировать alpha.domain.agent.xml для настройки доменного взаимодействия

**Особенности:**
- Файлы конфигурации критичны для сетевого взаимодействия
- При резервировании настроить соответствующие порты и адреса

**Источник:** Инструкция по настройке Domain.docx

## Как настроить PostgreSQL на RedOS

**Что нужно:**
- RedOS
- Официальный репозиторий PostgreSQL

**Шаги:**
1. Добавить официальный репозиторий PostgreSQL
2. Установить PostgreSQL 11: `yum install postgresql11-server`
3. Инициализировать базу данных: `postgresql-11-setup initdb`
4. Настроить автозапуск службы: `systemctl enable postgresql-11`
5. Запустить службу: `systemctl start postgresql-11`

**Особенности:**
- Используется именно PostgreSQL 11
- Требуется правильная инициализация базы

**Источник:** REDOS Установка PostgreSQL11.docx

## Диагностика: Как снять дамп процесса

**Что нужно:**
- Windows: Debug Diagnostic Tool или WinDbg
- Linux: gdb, Valgrind

**Шаги для Windows:**
1. Запустить Debug Diagnostic Tool
2. Выбрать режим "Native Memory and handle Leak"
3. Присоединиться к процессу Alpha Platform
4. Настроить сборщик дампа
5. Дождаться сбора данных или аварийного завершения

**Шаги для Linux:**
1. Найти PID процесса: `pgrep имя_процесса`
2. Присоединиться через gdb: `gdb -p PID`
3. Создать core-файл: `generate-core-file`
4. При блокировке ptrace: `astra-ptrace-lock disable` + перезагрузка

**Особенности:**
- На Astra Linux может быть включена блокировка трассировки ptrace
- Дампы занимают много места на диске

**Источник:** Инструкция по снятию дампа.docx, Снять дамп через WinDBG.docx, Инструкция по снятию дампа Astra (При включенной блокировке трассировки ptrace).txt
