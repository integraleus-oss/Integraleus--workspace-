## Часть 1. Диагностика текущего состояния

### Что есть сейчас
- Монолитный документ ~6500 стр. (PDF/Word)
- Покрытие: 50 модулей, все протоколы, справочники элементов и атрибутов
- Базовый учебный курс (отдельный документ, ~395K символов)
- Документ «Новинки 2024-2025»

### Ключевые проблемы
| # | Проблема | Влияние |
|---|----------|---------|
| 1 | Нет Quick Start — новичок тонет в 6500 страницах | Высокий порог входа, потеря потенциальных пользователей |
| 2 | Reference-стиль: описано «что есть», но не «как сделать» | Инженер не может решить задачу без звонка в техподдержку |
| 3 | Нет Troubleshooting | Техподдержка перегружена типовыми вопросами |
| 4 | Мало визуала (скриншоты, схемы, видео) | Сложно ориентироваться в интерфейсе |
| 5 | Нет веб-версии с поиском | Невозможно быстро найти нужное |
| 6 | Нет привязки к версиям | Путаница между Alpha.Server 6.2 и 6.4 |
| 7 | Единый документ для всех ролей | Оператор, проектировщик и руководитель читают одно и то же |

## Часть 3. Quick Start — детальная структура

### Quick Start: «От нуля до мнемосхемы за 30 минут»

**Целевая аудитория:** Инженер АСУ ТП, впервые открывший Альфа платформу.
**Результат:** Работающий проект с 5 сигналами, Modbus-симулятором и мнемосхемой.

---

#### Шаг 0. Что понадобится (1 мин)
- ПК с Windows 10/11 или Linux (Ubuntu/Astra)
- Дистрибутив Alpha Platform (ссылка на скачивание)
- USB-ключ лицензии или демо-лицензия (если есть)
- [Опционально] Modbus-симулятор (ModRSsim2 или встроенный Alpha.Imitator)

> 💡 **Совет:** Для знакомства можно использовать Alpha.Imitator — он уже входит в поставку.

#### Шаг 1. Установка (5 мин)
- Установка Alpha.DevStudio
- Установка Alpha.Server
- Установка Alpha.HMI
- Проверка: все три компонента запускаются
- ⚠️ **Чек-лист:** .NET 8.0 Runtime установлен? (для Linux: + libgdiplus)

> 📸 *Скриншот: окно установщика с отмеченными компонентами*

#### Шаг 2. Создание проекта в DevStudio (5 мин)
1. Запустить Alpha.DevStudio
2. Файл → Новое решение → имя «QuickStart»
3. Добавить проект «MyFirstProject»
4. Добавить компонент Alpha.Server
5. Добавить компонент Alpha.AccessPoint

> 📸 *Скриншот: дерево решения с двумя компонентами*

#### Шаг 3. Создание сигналов (5 мин)
1. В Alpha.Server создать 5 сигналов:
   - `Temperature` (float, -50…200 °C)
   - `Pressure` (float, 0…25 бар)
   - `Level` (float, 0…100 %)
   - `PumpRunning` (bool)
   - `ValvePosition` (uint1, 0…100 %)
2. Назначить адреса Modbus (HR 0–4)
3. Настроить мёртвую зону (deadband) для аналоговых

> 📋 *Таблица: имя сигнала → тип → адрес Modbus → описание*

#### Шаг 4. Подключение к источнику данных (5 мин)

**Вариант А: Alpha.Imitator (без реального оборудования)**
1. Добавить компонент Alpha.Imitator
2. Настроить генерацию значений (синус для Temperature, рампа для Level)
3. Привязать к сигналам

**Вариант Б: Modbus-симулятор**
1. Запустить ModRSsim2 (или аналог) на порту 502
2. В DevStudio: добавить Modbus TCP Master
3. Настроить IP: 127.0.0.1, порт 502
4. Привязать сигналы к Holding Registers

> 📸 *Скриншот: настройка Modbus-адаптера*

#### Шаг 5. Компиляция и запуск (3 мин)
1. Компиляция: Решение → Компилировать всё
2. Построение конфигурации: Развёртывание → Построить
3. Деплой: Развёртывание → Развернуть
4. Проверка: Alpha.Server запущен, сигналы обновляются

> 📸 *Скриншот: окно диагностики — сигналы с живыми значениями*

#### Шаг 6. Создание мнемосхемы в HMI (7 мин)
1. Открыть Alpha.HMI → Дизайнер
2. Создать новую форму «Обзор»
3. Добавить элементы:
   - Текстовое поле с привязкой к `Temperature` (цвет по условию: >100°C — красный)
   - Индикатор гистограммы для `Level`
   - Кнопка для `PumpRunning` (вкл/выкл)
   - Ползунок для `ValvePosition`
   - Индикатор прогресса для `Pressure`
4. Настроить источник данных: AP, порт 4388
5. Запустить Визуализатор

> 📸 *Скриншот: готовая мнемосхема с живыми данными*

#### Шаг 7. Что дальше? (1 мин)
Поздравляем! У вас работает:
- ✅ Alpha.Server с 5 сигналами
- ✅ Подключение к источнику данных
- ✅ Мнемосхема с визуализацией

**Следующие шаги:**
- 📘 [Настройка тревог](../howto/alarms-basic.md) — добавить аларм при Temperature > 150°C
- 📘 [Архивирование](../howto/historian-setup.md) — подключить Historian для записи истории
- 📘 [Веб-доступ](../howto/webviewer-setup.md) — открыть мнемосхему в браузере
- 📘 [Резервирование](../howto/hot-standby.md) — настроить отказоустойчивость

## Часть 6. Рекомендации по инструментам

### Движок документации
| Вариант | Плюсы | Минусы |
|---------|-------|--------|
| **Docusaurus** (React) | Версионирование, поиск Algolia, i18n, MDX | Нужен Node.js, React-экосистема |
| **MkDocs + Material** (Python) | Простота, отличный поиск, красивая тема | Меньше гибкости для кастомизации |
| **GitBook** (SaaS) | Нулевой деплой, WYSIWYG | Платный для команд, vendor lock |

**Рекомендация:** MkDocs + Material — оптимальный баланс простоты и возможностей. Русскоязычная SCADA-аудитория оценит быстрый поиск и чистый дизайн.

### Линтеры
- **Vale** — проверка стиля, терминологии, консистентности
- Создать словарь терминов Альфа платформы (сигнал, мнемосхема, квитирование и т.д.)

### Визуал
- **Draw.io / diagrams.net** — архитектурные схемы (бесплатный, экспорт в SVG)
- **Snagit / ShareX** — скриншоты с аннотациями
- **OBS / ScreenPal** — короткие видео (30-90 сек, GIF)

### Workflow
```
Автор (Markdown) → Git PR → Ревью → CI (Vale + build) → Деплой (docs.automiq.ru)
```
