## 2. Архитектура — принципиальное различие

### Альфа платформа
**Монолитная SCADA** — всё в одной платформе:
- Сбор данных с контроллеров (OPC, Modbus, IEC 104/101/61850, S7, FINS, EtherNet/IP и др.)
- Визуализация и управление (мнемосхемы HMI)
- Архивирование (Historian)
- Тревоги (Alarms)
- Отчёты (Reports)
- Безопасность (Security)

Оператор сидит перед мнемосхемой и **управляет** процессом.

### AVEVA PI System
**Платформа данных** — собирает, хранит, контекстуализирует и раздаёт данные:
- Подключается к СУЩЕСТВУЮЩИМ SCADA/DCS/PLC через интерфейсы
- Хранит историю (Data Archive)
- Добавляет контекст (Asset Framework)
- Считает аналитику (Asset Analytics)
- Визуализирует (PI Vision) — но это **мониторинг**, не управление

PI System **НЕ заменяет SCADA**. Он стоит рядом и собирает из неё данные.

> **Вывод:** Это продукты для РАЗНЫХ задач. Альфа — это SCADA, из которой управляют процессом. PI System — это историан + аналитика, который берёт данные из SCADA и делает их полезными для инженеров, аналитиков и руководства.

| **Скорость чтения** | Высокая (конкретные цифры — по запросу у вендора) | Миллионы значений/сек |
| **Хранение событий** | ✅ Да | ✅ Через Event Frames |

| **Кроссплатформенность** | ✅ Windows + Linux | ❌ Только Windows |

### ✅ Плюс Альфы:
- Работает на Linux (Astra, РЕД ОС, Альт, Ubuntu) — критично для импортозамещения
- Формат 4x с журналом транзакций — надёжность при сбоях

### ✅ Плюс PI System:
- Двухуровневое сжатие (exception + compression) — более интеллектуальная фильтрация шума ДО записи
- Десятилетия доказанной работы в enterprise-инсталляциях (Shell, BP, Pfizer и др.)

| **Шаблоны** | Пользовательские типы в HMI (аналог классов ООП) | AF Templates — шаблоны активов с наследованием и автообновлением |
| **Множественные представления** | ❌ Одно дерево | ✅ Несколько иерархий одновременно (по географии, по процессу, по оборудованию) |

Asset Framework — одна из сильнейших сторон PI. Это полноценная семантическая модель предприятия. Ничего подобного по мощности у Альфы нет.

### ✅ Плюс Альфы:
Для типовой SCADA задачи (мнемосхема + управление + архив) сложная модель активов избыточна. Альфа решает задачу проще.

## 15. Когда что выбирать

### Выбирай Альфу, если:
- Нужна SCADA для управления процессом (оператор + мнемосхемы + команды на ПЛК)
- Российское предприятие, требования импортозамещения
- Объект КИИ — нужен реестр Минцифры
- Нужна работа на отечественных Linux
- Бюджет ограничен
- Масштаб от малых объектов (One+, до 50k) до крупных (SCADA/Platform, миллионы тегов)

### Выбирай PI System, если:
- Нужен enterprise-историан для аналитики (НЕ для управления)
- Масштаб: сотни тысяч — миллионы тегов
- Нужна модель активов и цифровые двойники
- Нужна интеграция с BI/ML
- Мультисайтовая инфраструктура с облаком
- Нет ограничений по импортозамещению

### Они НЕ конкуренты в чистом виде:
В реальных проектах PI System часто работает РЯДОМ со SCADA (в том числе рядом с Альфой). SCADA управляет процессом, PI System собирает данные из SCADA для аналитики. Это разные слои одной архитектуры.
