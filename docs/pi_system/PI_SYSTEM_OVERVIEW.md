## Общая информация

AVEVA PI System (ранее OSIsoft PI) — промышленная платформа для сбора, хранения, контекстуализации и анализа данных реального времени. В 2021 OSIsoft куплена AVEVA (Schneider Electric).

**Позиционирование:** мировой стандарт для историанов в нефтегазе, энергетике, химии, фарме, горнодобыче.

**Установочная база:**
- 85% крупнейших нефтегазовых компаний
- 1000+ энергетических компаний
- 9 из 10 крупнейших горнодобывающих
- 24 из 25 крупнейших фармкомпаний

## 1. PI Data Archive

**Назначение:** Ядро PI System — time-series база данных промышленного класса.

### PI Points (теги)
- Основная единица — PI Point (tag). Каждый тег = один поток данных
- Атрибуты тега: имя, тип данных (float32, float64, int32, digital, string), единицы измерения, описание, источник
- Конфигурация через PI System Management Tools или Excel-надстройку

### Двухуровневое сжатие данных

#### Exception Reporting (на стороне интерфейса)
- Простой линейный тест — фильтрация «шума»
- Отсекает незначительные изменения ниже точности прибора
- Значение, прошедшее тест → **Snapshot** (текущее значение)
- Рекомендация: exception deviation чуть меньше точности датчика

#### Compression Testing (на стороне Data Archive)
- Алгоритм **Swinging Door Compression**
- Хранит только «значимые» точки, остальные воссоздаются линейной интерполяцией
- «Дверь на петлях» — пока новые точки попадают в коридор между прямыми от последней сохранённой точки, они НЕ записываются
- Значение, прошедшее тест → **Archive value**
- Результат: десятилетия данных в компактном виде

### Производительность
- Миллионы тегов на одном сервере
- Сотни тысяч значений/сек на запись
- Миллионы значений/сек на чтение
- Проприетарные архивные файлы (.arc), оптимизированные под time-series

### Архитектура хранения
- Snapshot Subsystem — текущие значения (кэш в памяти)
- Archive Subsystem — исторические данные (на диске)
- Архивные файлы растут автоматически, настраиваемый размер и ротация

## 2. PI Asset Framework (AF)

**Назначение:** Контекстный/семантический слой поверх сырых тегов. Превращает «Tag_12345» в «Насос К-301 / Давление на выходе».

### Основные сущности

**AF Database** — контейнер верхнего уровня (обычно = предприятие или площадка)

**AF Elements** — узлы иерархии:
- Завод → Цех → Линия → Агрегат → Компонент
- Можно строить несколько иерархий одновременно (по оборудованию, по географии, по процессу)

**AF Attributes** — свойства элементов:
- Статические: производитель, модель, дата ввода
- Динамические (PI Point Data Reference): привязка к тегу Data Archive
- Формульные: вычисляемые из других атрибутов
- Table Lookup: из внешних SQL-таблиц

**AF Templates** — шаблоны:
- Определяешь шаблон «Насос» с набором атрибутов один раз
- Создаёшь 500 элементов по шаблону
- Изменение шаблона → все экземпляры обновляются автоматически
- Поддержка наследования шаблонов

### Масштаб в реальных проектах
- Shell: 7.5M потоков, 100k расчётов/мин
- Devon Energy: 50k+ активов, 50+ шаблонов
- MOL: 21k шаблонов, 80B точек/год
- BP: цифровые двойники 40 морских платформ, 1M+ потоков

## 3. Asset Analytics

**Назначение:** Потоковые вычисления в реальном времени поверх AF.

### Как работает
- Привязываются к AF-элементам через шаблоны
- Визуальный редактор формул (Expression Editor) — без программирования
- Вычисления в реальном времени по мере поступления данных

### Возможности
- Арифметика, логика, условия (If/Then/Else)
- Агрегатные функции: TagAvg, TagMax, TagMin, TagTotal за период
- Предиктивные модели и what-if анализ
- Rollup — агрегация дочерних элементов
- **Backfill** — пересчёт по историческим данным при изменении формулы
- **Recalculate** — пересчёт по запросу

### Шаблонизация
- Один расчёт в шаблоне → автоматически ко всем экземплярам
- Результаты пишутся обратно в PI Data Archive как новые теги

## 4. Event Frames

**Назначение:** Захват и анализ дискретных событий с привязкой к временным окнам.

### Концепция
- Event Frame = именованное событие с временем начала и конца
- Автоматически привязывает все данные за период события
- Примеры: партия (batch), смена, простой, аварийный останов, переходный режим

### Создание
- Через Asset Analytics: правило «если T > 80°C в течение 5 мин → создать Event Frame "Перегрев"»
- Через PI AF SDK программно
- Вручную через PI System Explorer

### Шаблоны Event Frame
- Стандартизируют захватываемые атрибуты
- Вложенность: Event Frame «Смена» → «Партия 1», «Партия 2»

### Анализ
- Сравнение однотипных событий (best batch vs worst batch)
- Root Cause Analysis
- Статистика по продолжительности, количеству, частоте

## 5. Notifications

**Назначение:** Автоматические оповещения по условиям.

### Механизм
- Триггер: значение тега выходит за порог / сложное многопараметрическое условие
- Действие: email, SMS, создание Event Frame, вызов веб-сервиса

### Функции
- Шаблоны уведомлений (templatize)
- Подтверждение получения (acknowledgement)
- Эскалация: если никто не подтвердил за N минут → уведомить руководителя
- Поддержка сложных типов данных для инициации нарядов в CMMS/SAP

## 6. PI Vision

**Назначение:** Веб-визуализация реального времени.

### Архитектура
- Веб-приложение на IIS (Windows Server)
- Доступ через любой браузер, включая мобильные
- 500k+ пользователей по миру

### Возможности
- Drag-and-drop создание дисплеев
- Символы: графики (trends), таблицы, gauges, формы, фигуры
- **Multi-state symbols** — цвет/мигание по значению (визуальные алармы)
- Ad hoc trending — посмотреть историю любого тега на лету
- Коллекции активов — группировка однотипных объектов с фильтрацией
- Шаблоны дисплеев — один дисплей для всех насосов, переключение между ними

### Безопасность
- Windows Authentication / OpenID Connect (SSO)
- Разграничение доступа на уровне элементов и тегов
- Шифрование трафика

## 7. PI DataLink

**Назначение:** Надстройка для Microsoft Excel.

### Функции
- Данные из PI Server прямо в ячейки Excel
- Специальные функции: PISampDat, PICompDat, PIArcVal, PITimeDat
- Работает с тегами и AF-атрибутами
- Поддержка Event Frames — выборка данных по событиям
- Динамическое обновление при открытии
- Комбинация с VBA/макросами

### Применение
- Суточные/сменные отчёты
- Анализ трендов в Excel
- KPI-дашборды для менеджмента
- Sustainability-отчётность (расход энергии/воды)

## 8. PI Integrator for Business Analytics

**Назначение:** Подготовка данных PI для BI-инструментов и ML.

### Как работает
- Drag-and-drop для выбора и подготовки данных
- Cleansing: очистка аномалий, пропусков
- Shaping: выравнивание timestamp'ов, интерполяция, формирование таблиц
- Push-модель: новые данные → автоматическая передача в BI

### Куда выгружает
- SQL Server, Azure SQL
- Power BI, Tableau, Qlik
- ML-платформы (Python/R через SQL)
- AVEVA CONNECT (облако)

## 9. Edge Data Store

**Назначение:** Лёгкий агент для сбора данных на edge-устройствах.

### Ключевое
- Работает на IIoT-шлюзах, одноплатных компьютерах
- Windows и Linux
- Persistent local storage — продолжает собирать при потере связи
- Протоколы: OPC UA, Modbus TCP (без кода), OMF
- REST API для локальных приложений
- Передаёт в PI Server или AVEVA CONNECT

### Типичный сценарий
Удалённая скважина → Edge Data Store на шлюзе → буфер при потере связи → отправка в PI Server при восстановлении

## 10. PI Interfaces / PI Connectors / AVEVA Adapters

**Назначение:** Сбор данных из промышленных источников.

### Три поколения
1. **PI Interfaces** (legacy) — 225+ протоколов, Windows-only
2. **PI Connectors** — более новые, Windows
3. **AVEVA Adapters** — кроссплатформенные (Windows/Linux/Docker): OPC UA, MQTT, Modbus TCP

## 11. Developer Technologies (API)

### PI Web API — RESTful
- HTTP-методы для чтения/записи данных, метаданных, событий
- Кроссплатформенный (любой язык)
- Основной API для новых разработок

### PI AF SDK — .NET
- Объектно-ориентированный доступ к AF, аналитикам, Event Frames
- Высокая производительность
- Только Windows/.NET

### PI SQL Framework
- SQL-запросы к данным PI
- OLEDB/ODBC провайдеры
- Для аналитиков, привыкших к SQL

### PI SDK (legacy, deprecated)
- COM-based, заменяется PI AF SDK и PI Web API

## 12. PI Data Infrastructure + CONNECT

**Назначение:** Гибридная платформа edge-to-cloud.

### Состав
- Все on-prem компоненты PI System
- **AVEVA CONNECT** — облачная SaaS-платформа:
  - Облачное хранение и визуализация
  - Шеринг данных с партнёрами/подрядчиками
  - Интеграция с AI/ML
- **Aggregate Tag** — покупка тегов пулом, динамическое распределение между серверами

---

## Лицензирование

- Модель: **подписка через AVEVA Flex Credits**
- Основная единица: количество тегов
- Варианты: per-server или aggregate (пул тегов на группу серверов)
- PI Data Infrastructure = единая подписка на все компоненты
- Включает доступ к AVEVA CONNECT
