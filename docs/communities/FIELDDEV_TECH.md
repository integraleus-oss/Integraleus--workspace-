## Концепция: Industrial AI + Physical AI

### Определения (из статьи fielddev)

- **Industrial AI** — ИИ для промышленности: анализ данных с оборудования, предиктивная аналитика, оптимизация процессов, управление жизненным циклом. Живёт в серверах и БД.
- **Physical AI** — ИИ, встроенный в физические системы: роботы, сенсоры, актуаторы. Воспринимает физический мир и действует в нём. NVIDIA: 1.3 млрд ИИ-роботов к 2035.

### Три этапа развития (roadmap fielddev)

1. **Инженерный ИИ** (сейчас) — ИИ-агент учится решать задачи инженеров, работать с цифровыми инструментами
2. **Промышленный ИИ** — интеграция в системы управления производством, предиктивный контроль
3. **Physical AI** — замыкание физического контура, роботы + автономные системы

### Новая архитектура управления

Вместо документооборота (паспорт → сертификат → акт → устарел) → **цифровая сущность** с историей изменений, связями и формализованными требованиями. Алгоритм проверяет соответствие автоматически.

Формула: **практика → методика → норматив**

## Проекты (с fielddev.tech и GitVerse)

### 1. IntRequirements_SimSYS — Интеграция с платформой цифровых двойников
**GitVerse:** https://gitverse.ru/fielddev/IntRequirements_SimSYS
**Статус:** АКТИВНЫЙ (обновлён 16 часов назад)

**Цель:** Стандартизованный механизм интеграции ПО с платформой digital twin.

**Архитектура:**
```
Платформа ЦД ↔ Message-сервис (RabbitMQ/MQTT) ↔ Расчётные сервисы
```

**Базовый стандарт:** ГОСТ Р 58546–2019 (модель службы обмена сообщениями)

**Стек:** асинхронный обмен, протокол MQTT, открытый код, Linux + Windows

**Результаты:**
- Определена архитектура интеграции (платформа ЦД + message-сервис + сторонние сервисы)
- Определены форматы и структура сообщений
- Выбран стек (RabbitMQ, MQTT)
- В работе: настройка message-сервиса, доработка платформы ЦД, поиск разработчиков расчётных сервисов

**Участники:** Леонид Латнер (Нефтеавтоматика), Кирилл Страмнов (Хромос-Инжиниринг)

**Ключевой тезис:** Message-сервис = единая шина данных для обмена между ЦД, расчётными сервисами и системами управления производством (MES).
