# Alpha Licensing Playbook (Share-Safe)

Краткая версия для рабочих чатов без финансовых и внутренних деталей.

## Когда применять

Для вопросов по:
- Альфа / Alpha
- лицензированию
- расчёту / пресейлу / КП

## Обязательный процесс

1. Собрать входные параметры:
   - архитектура
   - клиенты
   - драйверы/протоколы
   - резерв
   - terminal/RDP

2. Сделать преселекцию семейства:
   - One+ / SCADA / Platform

3. Перед финальным расчётом выполнить сверку
   - policy
   - актуального тарифа

4. Сформировать минимум 2 варианта КП
   (обычно SCADA + Platform)

5. Явно перечислить допущения/неизвестные.

## Правила, которые нельзя пропускать

- SCADA с резервом = 2 серверные лицензии.
- Для Full-клиентов terminal/RDP обязательно уточняется
  (или пометка «уточняется при заказе ключей»).
- Обмен One+/SCADA с Platform по умолчанию считать внешним.
- Фокус расчёта: драйверы и протоколы (OPC/IEC/Modbus/S7 и т.д.).
- В итоге указывать год тарифа.

## Что уточнять перед финалом

- внешний/внутренний характер каждого обмена
- фактический профиль клиентских подключений
- наличие и тип резервирования по каждому серверному контуру
