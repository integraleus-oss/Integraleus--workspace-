# EDGE_CASES.md — Граничные кейсы и трактовки

Использовать при спорных расчётах, чтобы не допускать фантазий.

## 1) Теги и интеграции

1. One+/SCADA -> Platform:
- считать внешней интеграцией (по умолчанию OPC UA),
- теги учитывать как внешние у принимающей стороны.

2. Platform <-> Platform:
- по Alpha.Link — внутренние теги не считаются внешними;
- по OPC/UA, Modbus, IEC и др. — считать внешними у принимающего сервера.

3. Режимы входных данных:
- signals_only: только пересчёт DI/DO/AI/AO;
- tags_only: только вручную заданные внешние теги;
- combine: сигналы + вручную заданные внешние теги.

## 2) Информационная мощность

Коэффициенты:
- simple: DIx2 DOx2 AIx3 AOx1
- medium: DIx2 DOx5 AIx10 AOx1
- complex: DIx3 DOx10 AIx10 AOx15
- custom: задаётся явно

Примечание: порядок опроса — сначала сигналы, потом уровень информационной мощности.

## 3) Клиенты и WEB

1. Full-клиенты:
- всегда уточнять распределение terminal/RDP;
- если неизвестно — не блокировать расчёт, добавить примечание «уточняется при заказе ключей».

2. WEB:
- если >5 одновременных WEB — добавить рекомендацию выделенного WEB-сервера;
- при выделенном WEB-сервере добавлять WEB-PORTAL (для SCADA/Platform).

## 4) Historian и Reports

1. Historian:
- рассчитывать отдельно от внешних тегов системы;
- при задании в процентах: historian_tags = ceil(total_tags * percent);
- выбирать ближайшую верхнюю ступень;
- резерв Historian = отдельная лицензия (обычно x2).

2. Reports:
- либо «не требуется»,
- либо подбирать серверный и клиентский профиль RPT.

## 5) Финансовая выдача

Без отдельного разрешения:
- артикулы можно,
- построчные цены по артикулам нельзя.

Показывать:
1) Лицензии (без НДС)
2) SLA (без НДС)
3) НДС 22% на SLA
4) SLA (с НДС)
5) Итого к оплате
6) В т.ч. НДС 22%
