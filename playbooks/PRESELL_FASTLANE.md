# PRESELL_FASTLANE.md — Быстрый пресейл (80% кейсов за 1–2 итерации)

## Этап 1 — 4 стартовых вопроса

1. Заказчик
2. Площадка/объект
3. Система/проект
4. Срок реализации

## Этап 2 — Лицензионное ядро

5. Сигналы/теги:
- DI / DO / AI / AO
- или готовое число внешних тегов

6. Уровень информационной мощности:
- simple / medium / complex / custom

7. Режим расчёта:
- signals_only / tags_only / combine

8. Серверы и резерв:
- сколько серверов
- где резерв

9. Клиенты:
- Full / WEB / Terminal
- terminal/RDP распределение

10. Интеграции/протоколы:
- OPC/IEC/Modbus/S7/и т.д.
- внешний/внутренний характер обмена

11. Historian/Reports/SLA:
- Historian: да/нет (+ объём)
- Reports: да/нет
- SLA: BASE/STD/OPT/PRM

## Этап 3 — Расчётный алгоритм (обязательный)

1. Выбрать режим расчёта тегов (обычно `signals_only`).
2. Посчитать теги минимум в 2 сценариях:
   - базовый (simple/medium),
   - консервативный (medium/complex).
3. Подобрать серверный уровень по ближайшей верхней ступени тарифа.
4. Разложить минимум в 2 продуктовых варианта:
   - Вариант A: Alpha.SCADA,
   - Вариант B: Alpha.Platform (если рост/мультисервер/сложные интеграции).
5. One+ предлагать только для малого простого сценария.
6. Учесть резерв:
   - SCADA резерв -> x2 серверных лицензий того же уровня,
   - Platform -> проверка архитектуры и правила внешних тегов.
7. Подобрать клиентов по верхней ступени (CL-F / CL-RO*), учесть terminal/RDP.
8. WEB > 5 -> добавить примечание про выделенный WEB-сервер (+ WEB-PORTAL при необходимости).
9. Подобрать драйверы (или запросить вендор+модель и подобрать самостоятельно).
10. Учесть Historian / Reports / SLA.

## Этап 4 — Правила быстрых дефолтов

- Если не указана SLA -> BASE.
- Если нужен финальный коммерческий вариант -> предлагать минимум 2 опции (обычно SCADA + Platform).
- Для Platform явно писать: драйверы включены.
- Если неизвестно terminal/RDP -> пометка «уточняется при заказе ключей».

## Этап 4 — Формат ответа

1. Подтверждённые входные данные
2. Расчёт тегов и выбор верхних ступеней
3. Вариант A (рекомендованный)
4. Вариант B (альтернативный)
5. Финансовый блок:
   - Лицензии без НДС
   - SLA без НДС
   - НДС 22% на SLA
   - SLA с НДС
   - Итого к оплате
6. Риски/неизвестные
